# 尽职调查清单

## 工作流程

### 第一步：确定尽调范围

向用户询问：
- **目标公司**：名称、行业、商业模式
- **交易类型**：平台收购、补充收购、成长型股权、资本重组、分拆
- **交易规模/复杂程度**：决定尽调深度
- **关键关注点**：已知需优先处理的问题（客户集中度、监管、环保等）
- **时间表**：意向书/交割目标时间

### 第二步：生成工作流清单

根据行业特点，生成覆盖所有主要工作流的清单：

**财务尽调**
- 盈利质量（QoE）——收入和EBITDA调整
- 营运资金分析——标准化 vs. 实际
- 债务及类债务项目
- 资本支出（维护性 vs. 增长性）
- 税务结构和风险敞口
- 审计历史和会计政策
- 模拟调整（运行率、协同效应）

**商业尽调**
- 市场规模和增长（TAM/SAM/SOM）
- 竞争定位和市场份额
- 客户分析——集中度、留存率、NPS
- 定价能力和合同结构
- 销售管线和订单积压
- 市场推广效能

**法律尽调**
- 公司架构和组织架构图
- 重大合同（客户、供应商、合作伙伴）
- 诉讼历史和未决索赔
- 知识产权组合和保护
- 监管合规
- 雇佣协议和竞业限制

**运营尽调**
- 管理团队评估
- 组织架构和关键人物风险
- IT系统和基础设施
- 供应链和供应商依赖
- 设施和不动产
- 保险覆盖

**人力资源/人员尽调**
- 组织架构图和人员趋势
- 薪酬基准
- 福利和养老金义务
- 关键员工留存风险
- 文化评估
- 工会/劳动协议

**IT/技术尽调**（适用于科技型企业）
- 技术栈和架构
- 技术债务评估
- 网络安全态势
- 数据隐私合规（GDPR、CCPA、SOC2）
- 产品路线图和研发支出
- 可扩展性评估

**环境/ESG**（如适用）
- 环境负债
- 监管合规历史
- ESG风险和机遇

### 第三步：状态跟踪

对每项进行跟踪：

| 事项 | 工作流 | 优先级 | 状态 | 负责人 | 备注 |
|------|-----------|----------|--------|-------|-------|
| QoE报告 | 财务 | P0 | 待处理 | | |
| 客户访谈 | 商业 | P0 | 进行中 | | 10个中完成3个 |

状态选项：未开始 → 已请求 → 已收到 → 审查中 → 完成 → 风险标记

### 第四步：风险标记汇总

维护在尽调过程中发现的风险标记清单：
- 发现了什么
- 属于哪个工作流
- 严重程度（交易终止/重大/可控）
- 缓解措施或解决路径
- 对估值或交易条款的影响

### 第五步：输出

- 按工作流分标签页的Excel工作簿（默认）
- 汇总仪表板：各工作流完成百分比、未完成事项、风险标记
- 交易团队每周状态更新格式

## 行业特定补充

根据行业自动添加相关事项：
- **软件/SaaS**：ARR质量、队列分析、托管成本、SOC2
- **医疗健康**：监管审批、报销风险、支付方构成
- **工业**：设备状况、环境修复、安全记录
- **金融服务**：监管资本、合规历史、信贷质量
- **消费品**：品牌健康度、渠道构成、季节性、库存管理

## 重要说明

- 优先处理P0事项——这些是意向书或交割的前置条件
- 标记卖方响应缓慢的事项——可能暗示问题
- 将数据室内容与清单交叉对照以识别缺失
- 随着尽调推进更新清单——这是一份动态文档
