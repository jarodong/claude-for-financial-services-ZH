# 投资组合AI就绪度评估

## 工作流程

### 第一步：连接投资组合数据

首先询问用户投资组合材料的存放位置。不要假设——提供选项：

- **MCP服务器** — 数据室、SharePoint、Google Drive，或已连接的投资组合运营数据库
- **本地文件** — 磁盘上包含季度演示文稿、财务数据、董事会材料的文件夹路径
- **文件上传** — 将PDF、PowerPoint或Excel直接拖拽到对话中

连接后，提取投资组合（或其子集）的季度更新、董事会演示文稿和财务数据。对每家公司，提取：行业、收入、按职能划分的人员数量、已提及的技术栈，以及已启动的AI/自动化项目。

如果用户只提供一家公司，仍执行扫描但跳过跨投资组合排序。

如果材料中不明确，提前询问：
- 每家公司的剩余持有期（距退出12个月以内时AI投资回报的重要性降低）
- 是否有被投企业已经部署了成功的项目

### 第二步：逐企业扫描

对每家公司，回答三个准入问题。三个都是"是"则**通过**。任何一个为"否"则**暂缓**，并注明解锁条件。

1. **数据是否就绪？** 他们能否为该用例提供干净的输入数据——客户列表、发票流、合同库——而无需先进行6个月的数据治理项目？
2. **是否有负责人？** 管理团队中有人会推动此事，而不是仅"支持"的赞助人。
3. **能否在30天内试点？** 一个团队、一个工作流、现成工具。如果回答以"首先我们需要……"开头，那就不是速赢。

然后识别前2-3个杠杆点。在成本结构和运营中寻找以下模式：

**后台部门（通常试点最快）**
- 发票处理、应付/应收账款匹配、费用分类
- 合同摘要——供应商协议、租赁、客户主服务协议
- 月末结账：对账、波动说明、贷方报告初稿

**收入/前台部门**
- RFP和提案初稿——如果收入基于项目制则杠杆效应大
- 销售电话摘要和CRM数据维护
- 客户支持工单分类和首次回复起草
- 配置型/复杂产品的报价生成

**运营（因行业而异）**
- SOP和质量文档生成
- 排班和调度（现场服务、物流）
- 代码生成和审查（软件类被投企业）

对每个杠杆点，用一行说明：替代什么、每周节省的全职人力工时（按30-50%估算，而非100%），以及是购买现成产品还是需要轻量开发。

### 第三步：跨投资组合排序

将每家公司的每个杠杆点汇总为一个清单。按以下标准排序：

1. **金额影响** — 年化EBITDA贡献（降本+增收，扣除工具成本）
2. **价值实现速度** — 到首个可衡量结果的月数
3. **概率** — 根据数据质量、变革管理风险、管理团队能力进行折减

决胜标准：优先考虑剩余持有期<18个月的机会——这些需要立即行动或放弃。

输出排序表：

| 排名 | 公司 | 机会 | 预估EBITDA（人民币） | 价值实现月数 | 准入状态 | 首要步骤 |
|---|---|---|---|---|---|---|
| 1 | | | | | 通过 | |
| 2 | | | | | 通过 | |
| 3 | | | | | 暂缓——[阻碍因素] | |

### 第四步：寻找可复制的打法

投资组合中最高杠杆的动作是在多家公司运行同一成功打法。扫描以下情况：

- **同行业、同职能** — 两家医疗服务类被投企业都有人工预授权流程？一次实施，两处部署。
- **同一工具、不同公司** — 如果一家被投企业已有运行良好的发票处理系统，将其他应付账款规模超过X百万元的被投企业标记为快速跟随者。
- **共享供应商议价** — 三家被投企业购买同一工具就是一次价格谈判。

列出每个可复制打法及其主导公司（验证方）和跟随公司（复制方）。

### 第五步：输出

为运营合伙人准备一页纸，按投资组合回顾的结构组织：

1. **投资组合前5名** — 第三步的排序表，包含负责人和30天首要步骤
2. **可复制打法** — 2-3个可同时覆盖多家公司的打法
3. **各公司通过/暂缓状态** — 每家一行；暂缓的说明解锁条件
4. **不做事项** — 纸面看起来不错但未通过准入的机会；避免运营合伙人每季度重新讨论
5. **汇总EBITDA贡献** — 整个投资组合的AI机会总额，分第1年速赢和第2-3年扩展

## 重要说明

- **按金额排序，而非按兴奋度排序。** 在一家4000万元收入的公司节省40万元的应付账款自动化，每次都比炫目的客户聊天机器人更有价值。
- **约束条件几乎总是数据，而非模型。** 如果公司无法提供干净的客户列表，AI就不是第一个项目——数据治理才是。要直言不讳。
- **现成工具优先。** 对于没有工程深度的公司，定制开发缓慢、昂贵且脆弱。优先选择可购买和部署的工具。
- **负责人是真正的准入条件。** 没有内部负责人的速赢会在90天内死亡。如果管理团队中没有人想要它，无论金额多大，都标记为暂缓。
- **持有期决定紧迫性。** 距退出3年的公司可以承担基础数据项目。距退出12个月的公司需要能在CIM中体现为LTM EBITDA的成果——否则跳过。
- **失败的试点是信号。** 如果管理层已尝试过但未持续，在再次提议同样的事情之前先查明原因。
