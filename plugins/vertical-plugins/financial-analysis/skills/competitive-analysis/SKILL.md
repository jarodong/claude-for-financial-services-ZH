# 竞争格局图谱

构建完整的竞争分析 Deck。这是一个两阶段流程：先收集需求并获得大纲批准，然后构建。

## 环境检查

此 Skill 在 PowerPoint 加载项和聊天中均可工作。开始前识别你所在的环境——机制不同，工作流相同：

- **加载项** —— Deck 实时打开；直接在其中构建幻灯片。
- **聊天** —— 生成 `.pptx` 文件（或构建到用户上传的文件中）。

以下内容适用于两种环境。

## 阶段 1 —— 界定分析范围

竞争分析对不同人意味着不同的东西。在任何研究或幻灯片构建之前，使用 `ask_user_question` 确定他们实际想要什么。不要猜测——20 页的同行对标 Deck 和 5 页的市场图谱都是"竞争分析"，但形状完全不同。

如果可以，一轮收集（工具最多 4 个问题）：

- **范围** —— 单一目标公司周围有竞争对手？还是多公司并排无主角？
- **竞争对手集** —— 哪些公司在范围内？如果用户命名了，使用 exactly 那些。如果他们说"老样子"，提出一组并确认。
- **受众和深度** —— 给已经在行业里的人快速阅读，还是完整入门？这决定你需要市场规模、行业经济学和历史——还是比较。
- **投资背景** —— 他们需要 Bull/Base/Bear 情景和路标吗？那是下面的第 9 步；如果是战略审查而非投资论点则跳过。

如果他们上传了含竞争对手数据的 Excel/CSV，在开始提取数字前确认哪列映射到哪个指标。源文件保真度重要：按给定值原样使用，不重新计算或重新四舍五入。

## 阶段 2 —— 大纲、批准、然后构建

**在大纲批准前不要创建幻灯片。** 提出幻灯片标题和单行内容说明，展示给用户，获得同意。竞争 Deck 是 10-20 页相互关联的内容——因为第 4 页错误而重建代价很高。大纲是廉价的迭代点。

提出大纲时，`ask_user_question` 适用于结构性决策：使用哪种定位可视化（2×2 矩阵/雷达/层级图——下面第 5 步），如何分组竞争对手（按商业模式/细分/姿态——第 4 步）。这些是用户可能有偏好的品味决策。

## 标准——贯穿始终

### 提示保真度

当用户指定某事时，那是要求，不是建议：
- **幻灯片标题和章节名称** —— 措辞精确。如果他们说"Overview and Competitive Scope"，不要换成"FY2024 Competitive Landscape"。
- **图表 vs 表格** —— 不可互换。"Embedded chart"意味着带数据标签的真实图表对象，不是格式化表格。
- **完整数据系列** —— 如果他们列出 7 个竞争对手，包含全部 7 个。如果显示 2015-2025，包含每一年。
- **精确值和比率** —— "surpasses DoorDash 4:1, Lyft 8:1"意味着那些比率，不是"7.6x Lyft"。

### 来源质量，来源冲突时

1. 年报（审计）
2. 财报电话会 / 投资者演示（管理层评论）
3. 卖方研究（分析师估计，对私营公司规模有用）
4. 行业报告（McKinsey、Gartner——市场规模、趋势）
5. 新闻（仅近期发展；对照原始来源验证）

### 数据可比性

- 所有竞争对手指标来自同一财年；明确标记例外（"FY24" vs "H1 2024"）
- 跨竞争对手相同指标定义
- 国际数据转换为 USD；注明汇率和日期
- 缺失数据显示为"-"或"N/A"，估计值加"[E]"标记——永不空白
- 每个数字有引用："[公司] [文档] ([日期])"

### 设计

- **幻灯片标题是洞察，不是标签。** "Scale leaders pulling away from niche players"——不是"Competitive Analysis"。
- **路标量化。** "Margin below 40%"——不是"margins decline"。
- **评级显示实际。** "●●● 16000亿元"——不只是"●●●"。
- **图表是真实图表对象**——不是伪装成图表的文本表格。

**排版** —— 显式设置，不依赖默认值：
- 幻灯片标题：28-32pt 粗体
- 章节标题：18-20pt 粗体
- 正文：14-16pt（不低于 14pt）
- 表格文本：14pt
- 来源/脚注：14pt，灰色
- 相同元素类型 = 全 Deck 相同大小

**图表：**
- 图例在图表边界内，不浮动在绘图区域上
- 饼图右侧图例（≤6 个切片），折线/柱形图底部图例（≤4 个系列）
- 超过 6 个系列 → 拆分为多个图表或使用表格
- 饼图在切片上显示百分比，不只在图例中

**表格：**
- 浅灰色标题行，粗体
- 数字右对齐，文本左对齐
- 足够的单元格内边距使文本不触碰边框

**颜色：** 最多 2-3 种颜色。柔和——海军蓝、灰色、一个强调色。全 Deck 相同颜色含义。

### 严格 vs 灵活

| 姻终 | 视情况 |
|------|--------|
| 用户指定时精确标题/章节 | 未指定时创意标题 |
| 用户说图表时图表；说表格时表格 | 未指定时可视化类型 |
| 他们列出的每个竞争对手/数据点 | 未指定时竞争对手数量 |
| 指定时精确值 | 精度未指定时四舍五入 |
| 标题无溢出 | 竞争对手类别数 |
| 无重叠元素 | 比较维度 |

## 分析工作流

### 第 0 步 —— 行业定义指标

首先：这个行业实际运行的 3-5 个指标是什么？跨每个竞争对手一致使用。

| 行业 | 关键指标 |
|------|----------|
| SaaS | ARR、NRR、CAC 回收期、LTV/CAC、Rule of 40 |
| 支付 | GPV、费率、附加率、交易利润率 |
| 市场 | GMV、费率、买方/卖方比率、复购率 |
| 零售 | 同店销售、存货周转、每平方英尺销售 |
| 物流 | 量、单位成本、准时交付率、产能利用率 |

行业未列出——选择投资者和运营者对标的指标。

### 第 1 步 —— 市场背景

规模、增长、驱动因素、逆风。附来源。

正确："嵌入式支付 2024 年市场规模 8000-10000亿元，增速 20-25% CAGR（McKinsey 2024）"
错误："The market is large and growing rapidly"

### 第 2 步 —— 行业经济学

映射价值如何流动。方法取决于行业结构：
- **垂直结构** —— 价值链层级，每层典型利润率
- **平台/网络** —— 生态参与者，之间价值流动
- **碎片化** —— 整合动态，按规模利润率差异

### 第 3 步 —— 目标公司概况

```
| 指标 | 值 |
|------|-----|
| Revenue | 49.6亿元 |
| Growth | +26% YoY |
| Gross Margin | 45% |
| Profitability | 3.73亿元 Adj. EBITDA |
| Customers | 134K |
| Retention | 92% |
| Market Share | ~15% |
```

多分部公司加分部明细：

```
| 分部 | Revenue | Rev YoY | Rev % | EBITDA | EBITDA YoY | Margin |
|------|---------|---------|-------|--------|------------|--------|
| Seg A | 251亿元 | +26% | 57% | 65亿元 | +31% | 26% |
| Seg B | 138亿元 | +31% | 31% | 25亿元 | +64% | 18% |
| Seg C | 51亿元 | -2% | 12% | -0.74亿元 | -16% | -1% |
| Total | 440亿元 | +18% | 100% | 65亿元* | - | 15% |
```
*注：如适用，含公司成本

### 第 4 步 —— 竞争对手映射

按适合的视角分组（如果用户未指定，这是好的 `ask_user_question` 决策）：
- 按商业模式 —— 平台 / 垂直 / 水平
- 按细分 —— 企业 / SMB / 消费者
- 按姿态 —— 直接 / 相邻 / 新兴
- 按来源 —— 在位者 / 颠覆者 / 新进入者

### 第 5 步 —— 定位可视化

| 类型 | 适用场景 |
|------|----------|
| 2×2 矩阵 | 两个主导竞争因素 |
| 雷达/蛛网 | 多因素比较 |
| 层级图 | 自然聚类为战略群组 |
| 价值链图 | 垂直行业 |
| 生态系统图 | 平台市场 |

参见 `references/frameworks.md` 了解按行业的 2×2 轴对。

### 第 6 步 —— 竞争对手深度分析

每个竞争对手两张表。

**指标：**
```
| 指标 | 值 |
|------|-----|
| Revenue | X.X亿元 |
| Growth | +XX% YoY |
| Gross Margin | XX% |
| Market Cap | X.X亿元 |
| Profitability | XXX百万元 EBITDA |
| Customers | XXK |
| Retention | XX% |
| Market Share | ~XX% |
```

**定性：**
```
| 类别 | 评估 |
|------|------|
| Business | 做什么（1 句话） |
| Strengths | 2-3 条 |
| Weaknesses | 2-3 条 |
| Strategy | 当前优先事项 |
```

### 第 7 步 —— 比较分析

```
| 维度 | 公司 A | 公司 B | 公司 C |
|------|--------|--------|--------|
| Scale | ●●● 16000亿元 | ●●○ 4500亿元 | ●○○ 800亿元 |
| Growth | ●●○ +26% | ●●● +35% | ●●○ +22% |
| Margins | ●●○ 7.5% | ●○○ 3.2% | ●●● 15% |
```

### 第 8 步 —— 战略背景

M&A 交易（倍数、逻辑）、合作趋势、融资模式、监管动态。参见 `references/schemas.md` 了解 M&A 交易表格式。

### 第 9 步 —— 综合

**护城河评估** —— 对每个竞争对手评级 Strong / Moderate / Weak：

| 护城河 | 评估内容 |
|--------|----------|
| 网络效应 | 用户/供应商飞轮强度；跨边 vs 同边 |
| 转换成本 | 技术集成深度、合同锁定、行为习惯 |
| 规模经济 | 体量下的单位成本优势；最小有效规模 |
| 无形资产 | 品牌、专有数据、监管牌照、专利 |

**必需综合要素：**
- 持久优势（难以复制）——映射到护城河类别
- 结构性弱点（难以修复）
- 现状 vs 轨迹

**投资背景**（如阶段 1 范围说不需要则跳过）：

```
| 情景 | 概率 | 关键驱动 |
|------|------|----------|
| Bull | 30% | 市场份额增长、利润率扩张 |
| Base | 50% | 当前轨迹继续 |
| Bear | 20% | 竞争压力、利润率压缩 |
```

## 质量检查清单

完成前检查：

**提示保真度**
- 幻灯片标题与用户指定的完全一致
- 他们说图表时是图表；说表格时是表格
- 他们列出的每个竞争对手/年份/数据点都存在
- 指定时的精确值和格式

**数据一致性**
- 源文件值直接提取，不重新计算
- 同一指标在出现的每张幻灯片上显示相同值
- 与源相同的十进制精度

**布局**
- 标题无溢出
- 无重叠元素
- 所有文本在容器内，无裁剪

**内容**
- 每个数字有引用
- 所有指标来自同一财年（或已标记）
- 幻灯片标题陈述洞察，不是主题
- 图表是真实图表对象

对每张幻灯片运行标准视觉验证检查——这会捕捉你在读回 XML 时不显示的重叠、溢出和低对比度文本。
