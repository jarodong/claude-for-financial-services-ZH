# 清理数据

清理活动工作表或指定范围中的杂乱数据。

## 环境

- **如果在 Excel 中运行（Office Add-in / Office JS）：** 直接使用 Office JS（`Excel.run(async (context) => {...})`）。通过 `range.values` 读取，通过 `range.formulas = [["=TRIM(A2)"]]` 写入辅助列公式。原地 vs 辅助列的决策仍然适用。
- **如果操作独立 .xlsx 文件：** 使用 Python/openpyxl。

## 工作流

### 第一步：确定范围

- 如果给定范围（如 `A1:F200`），使用它
- 否则使用活动工作表的整个已用范围
- 分析每列：检测其主要类型（文本/数字/日期）并识别异常值

### 第二步：检测问题

| 问题 | 关注内容 |
|------|----------|
| 空格 | 前导/尾随空格、双空格 |
| 大小写 | 分类列中不一致的大小写（`usa` / `USA` / `Usa`） |
| 数字存为文本 | 数值存储为文本；数字单元格中多余的 `$`、`,`、`%` |
| 日期 | 同一列中混合格式（`3/8/26`、`2026-03-08`、`March 8 2026`） |
| 重复 | 完全重复行和近似重复（大小写/空格差异） |
| 空白 | 有数据列中的空单元格 |
| 混合类型 | 98% 是数字但有 3 个文本条目的列 |
| 编码 | 乱码（`Ã©`、`â€™`）、不可打印字符 |
| 错误 | `#REF!`、`#N/A`、`#VALUE!`、`#DIV/0!` |

### 第三步：提出修复方案

在修改前显示摘要表：

| 列 | 问题 | 数量 | 拟修复方案 |
|----|------|------|------------|

### 第四步：执行

- **优先使用公式而非硬编码清理值** —— 当清理后的输出可以表示为公式时（如 `=TRIM(A2)`、`=VALUE(SUBSTITUTE(B2,"$",""))`、`=UPPER(C2)`、`=DATEVALUE(D2)`），在相邻辅助列中写入公式，而非在 Python 中计算结果后覆盖原始值。这保持了转换的透明性和可审计性。
- 仅在用户明确要求或没有合理的公式等效时（如编码/乱码修复）才原地覆盖计算值
- 对于破坏性操作（去重、填充空白、覆盖原值），先与用户确认
- 每类修复后（空格 → 大小写 → 数字转换 → 日期 → 去重），向用户展示变更样本并获得确认后再进行下一类
