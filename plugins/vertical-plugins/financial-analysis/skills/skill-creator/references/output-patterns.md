# 输出模式

当技能需要产生一致、高质量的输出时，使用以下模式。

## 模板模式

为输出格式提供模板。根据需求匹配严格程度。

**严格要求（如 API 响应或数据格式）：**

```markdown
## 报告结构

始终使用此精确的模板结构：

# [分析标题]

## 执行摘要
[关键发现的单段概述]

## 主要发现
- 发现 1 及支撑数据
- 发现 2 及支撑数据
- 发现 3 及支撑数据

## 建议
1. 具体可执行的建议
2. 具体可执行的建议
```

**灵活指导（需要适应性时）：**

```markdown
## 报告结构

以下是合理的默认格式，请根据判断调整：

# [分析标题]

## 执行摘要
[概述]

## 主要发现
[根据发现内容调整各节]

## 建议
[根据具体情境定制]

根据分析类型需要调整各节。
```

## 示例模式

对于输出质量依赖于示例的技能，提供输入/输出配对：

```markdown
## 提交信息格式

按以下示例生成提交信息：

**示例 1：**
输入：添加了基于 JWT 令牌的用户认证
输出：
```
feat(auth): 实现基于 JWT 的认证

添加登录端点和令牌验证中间件
```

**示例 2：**
输入：修复了报告中日期显示不正确的 bug
输出：
```
fix(reports): 修正时区转换中的日期格式

在报告生成中统一使用 UTC 时间戳
```

遵循此风格：type(scope): 简要描述，然后详细说明。
```

示例比描述更能帮助 Claude 理解所需的风格和详细程度。
