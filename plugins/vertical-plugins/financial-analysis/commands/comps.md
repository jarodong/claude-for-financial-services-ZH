# 可比公司分析 Command

构建机构级可比公司分析，含运营指标、估值倍数和统计基准。

## 工作流

### 第一步：收集公司信息

如果提供了公司名称或代码，直接使用。否则询问：
- "你想分析哪家公司？"

### 第二步：加载 Comps 分析 Skill

使用 `skill: "comps-analysis"` 构建分析：

1. **明确分析目的**：
   - "核心问题是什么？"（估值、效率、增长比较）
   - "受众是谁？"（投决会、董事会、快速参考）
   - "你有偏好的格式或模板吗？"

2. **确定可比组**（4-6 家可比公司）：
   - 相似商业模式
   - 相似规模/市值区间
   - 同行业/同板块
   - 地域可比性

3. **收集数据**（优先使用 MCP 数据源）：
   - 运营指标：Revenue、Growth、Gross Margin、EBITDA、EBITDA Margin
   - 估值：Market Cap、EV、EV/Revenue、EV/EBITDA、P/E
   - 根据行业补充指标（SaaS 用 Rule of 40 等）

4. **构建分析**：
   - 运营统计部分：公司数据 + 统计值（Max、75th、Median、25th、Min）
   - 估值倍数部分：同样含统计摘要
   - 注释与方法论文档

### 第三步：生成 Excel 输出

生成 Excel 文件，包含：
- 标题区（分析标题、公司列表、日期、单位）
- 运营统计与财务指标部分
- 估值倍数部分
- 每个指标的统计摘要
- 注释部分，记录数据来源和方法论

### 第四步：交付输出

提供：
1. **Excel 文件**（.xlsx）—— Comps 分析
2. **摘要**，重点说明：
   - 可比组选择理由
   - 关键发现（谁溢价/折价交易）
   - 中位数倍数供参考

## 行业特定指标

| 行业 | 补充指标 |
|------|----------|
| 软件/SaaS | ARR、Net Dollar Retention、Rule of 40 |
| 零售 | 同店销售、存货周转 |
| 金融 | ROE、ROA、效率比率 |
| 制造 | 资产周转率、CapEx/Revenue |
| 医疗 | R&D/Revenue、管线价值 |

## 质量检查清单

交付前检查：
- [ ] 4-6 家真正可比的公司
- [ ] 时间区间一致（全部 LTM 或全部 FY）
- [ ] 所有公式引用单元格（无硬编码值）
- [ ] 所有硬编码输入的单元格注释含数据来源
- [ ] 统计值包含 Max、75th、Median、25th、Min
- [ ] 注释部分记录数据来源和方法论
- [ ] 蓝色 = 输入，黑色 = 公式
- [ ] 合理性检查通过（利润率逻辑、倍数合理）
