# 财务数据包构建器

为私募股权、投行和资管业务构建专业、标准化的财务数据包。将来自 CIM、发售备忘录、上市公司公告文件、网络搜索或 MCP 服务器的财务数据转化为可直接用于投资委员会审查的 Excel 工作簿。

**重要提示：** 在整个工作流程中，使用 xlsx 技能进行所有 Excel 文件的创建和操作。

## 关键成功要素

每个数据包必须达到以下标准。任何一项不达标都将导致交付物不可用。

### 1. 数据准确性（零容错）
- 每个数字均可追溯至源文件并注明页码
- 全部使用公式计算（无硬编码值）
- 交叉核对小计和合计的内部一致性
- 验证资产负债表平衡：资产 = 负债 + 所有者权益
- 确认现金流与资产负债表变动一致

### 2. 核心规则

**规则一：财务数据（计量货币） → 货币格式带 ¥ 符号**
触发词：Revenue, Sales, Income, EBITDA, Profit, Loss, Cost, Expense, Cash, Debt, Assets, Liabilities, Equity, Capex
格式：百万用 ¥#,##0.0，千用 ¥#,##0
负数：¥(123.0) 而非 -¥123

**规则二：运营数据（计数） → 数字格式，不带 ¥**
触发词：Units, Stores, Locations, Employees, Customers, Square Feet, Properties, Headcount
格式：#,##0 带千位分隔符
负数：(123) 与表格其余部分保持一致

**规则三：百分比（比率和比例） → 百分比格式**
触发词：Margin, Growth, Rate, Percentage, Yield, Return, Utilization, Occupancy
格式：0.0% 保留一位小数
显示：15.0% 而非 0.15

**规则四：年份 → 文本格式以防止插入千位分隔符**
格式：文本或自定义格式防止显示为 2,024
显示：2020, 2021, 2022, 2023A, 2024E

**规则五：当上下文混合时，每个指标使用各自适当的格式**
示例：
```
板块分析, 2022, 2023, 2024
零售收入, ¥50.0, ¥55.0, ¥60.0
  门店数, 100, 110, 120
  单店收入, ¥0.5, ¥0.5, ¥0.5
```
收入和单店指标使用 ¥，门店数使用数字格式。

**规则六：所有计算使用公式 → 永远不要硬编码计算值**
所有小计、合计、比率和衍生指标必须基于公式，而非硬编码值。这确保准确性并支持动态更新。

### 3. 专业展示标准

**格式标准：**

**配色方案 - 两层：**

**第一层：字体颜色（xlsx 技能强制要求）**
- **蓝色文字（RGB: 0,0,255）**：所有硬编码输入（历史数据、假设），非常规文字
- **黑色文字（RGB: 0,0,0）**：所有公式和计算
- **绿色文字（RGB: 0,128,0）**：链接至其他工作表

**第二层：填充颜色（可选，增强展示效果）**
- 填充颜色为可选项，仅在用户要求或需要增强展示时使用
- 如用户要求颜色或专业格式，使用以下标准配色：
  - **章节标题**：深蓝（RGB: 68,114,196）背景配白色文字
  - **子标题/列表头**：浅蓝（RGB: 217,225,242）背景配黑色文字
  - **输入单元格**：浅绿/米色（RGB: 226,239,218）背景配蓝色文字
  - **计算单元格**：白色背景配黑色文字
- 用户可指定自定义品牌颜色覆盖

**两层如何配合使用（如使用填充颜色）：**
- 输入单元格：蓝色文字 + 浅绿填充 = "用户输入数据"
- 公式单元格：黑色文字 + 白色背景 = "计算值"
- 工作表链接：绿色文字 + 白色背景 = "引用自其他标签页"

**字体颜色告诉你它是什么。填充颜色告诉你它在哪里（如使用）。**

**重要提示：** xlsx 技能的字体颜色为强制要求。填充颜色为可选项 —— 默认白色/无填充，除非用户要求增强格式或配色。

**始终应用：**
- 粗体标题，左对齐
- 数字右对齐
- 子项缩进2个空格
- 小计上方单下划线
- 最终合计下方双下划线
- 冻结行/列表头
- 最小边框（仅在结构需要处）
- 统一字体（通常 Calibri 或 Arial 11pt）

**绝不包含：**
- 每个单元格都加边框
- 多种字体或字号
- 除非特别要求，不使用图表
- 过度格式化或装饰

## 结构一致性
除非另有明确指示，使用标准8个标签页结构：
1. 执行摘要
2. 历史财务数据（利润表）
3. 资产负债表
4. 现金流量表
5. 运营指标
6. 物业/板块表现（如适用）
7. 市场分析
8. 投资亮点

### 标签页1：执行摘要
用途：为繁忙的高管提供一页概述

内容：
- 公司概述（商业模式2-3句话）
- 核心投资亮点（3-5个要点）
- 财务快照表（最近3年 + 预测的收入、EBITDA、增长率）
- 交易概况（如适用）
- 关键指标突出展示

格式：简洁、粗体标题、最少装饰、关键数字突出

### 标签页2：历史财务数据（利润表）
用途：完整的利润与亏损历史

内容：
- 按板块/产品线分列的收入
- 销售成本/营收成本
- 毛利润和毛利率 %
- 详细运营费用（S&M, R&D, G&A）
- EBITDA 和调整后 EBITDA
- 线下项目（D&A, 利息, 税项）
- 净利润

格式：
- 年份作为列（文本格式：2020, 2021, 2022）
- 百万元或千元（在顶部明确标注单位）
- 所有财务数据使用会计格式
- 小计上方单下划线，净利润下方双下划线
- 所有数字右对齐

### 标签页3：资产负债表
用途：期末财务状况

内容：
- 流动资产（现金、应收账款、存货、预付、其他）
- 长期资产（固定资产、无形资产、商誉、其他）
- 流动负债（应付账款、应计费用、短期债务、其他）
- 长期负债（长期债务、递延税项、其他）
- 股东权益（普通股、留存收益、其他）

格式：
- 验证公式：资产 = 负债 + 所有者权益
- 统一日期标注
- 包含营运资金计算
- 主要小计上方单下划线，最终合计下方双下划线

### 标签页4：现金流量表
用途：现金产生与使用分析

内容：
- 经营活动现金流（优选间接法）
- 投资活动现金流（资本支出、收购、资产出售）
- 融资活动现金流（发行/偿还债务、股权、分红）
- 现金净变动
- 期初和期末现金余额

格式：
- 尽可能链接至利润表和资产负债表
- 展示净利润到经营活动现金流的调节
- 清晰标注现金使用（流出）与来源（流入）

### 标签页5：运营指标
用途：非财务 KPI 和运营数据

内容（因行业而异）：
- 单位数量、客户数量、门店数
- 生产力指标（人均收入、单店收入、单位收入）
- 产能利用率
- 市场份额
- 客户留存/流失率
- 行业特定 KPI

**关键格式说明：**
运营指标不使用货币符号。这些是数量，不是货币。

格式：
- 明确标注单位（客户、员工、门店、平方英尺等）
- 整数带千位分隔符：1,250 而非 ¥1,250
- 比率使用百分比：95.0%
- 数字右对齐

### 标签页6：物业/板块表现（如适用）
用途：按业务单元、物业或板块的详细分列

内容：
- 按板块分列的收入和盈利能力
- 按地点/产品分列的关键指标
- 板块特定 KPI
- 比较绩效分析

格式：收入/EBITDA 与财务标签页一致，运营指标使用数字格式

### 标签页7：市场分析
用途：行业背景和竞争定位

内容：
- 市场规模和增长趋势
- 竞争格局概述
- 市场份额分析
- 行业基准和同行比较
- 相关监管环境

格式：叙述文本与表格混合，注明市场数据来源

### 标签页8：投资亮点
用途：核心投资论点的叙述性总结

内容：
- 竞争优势详细阐述
- 增长机遇和战略举措
- 风险因素和缓释策略
- 管理层评估和业绩记录
- 投资论点总结

格式：清晰标题、要点式、简洁段落

## 分步工作流程

### 阶段一：文档处理与数据提取

**步骤1.1：分析源数据**
- 获取源材料：上传的文档、公开文件的网络搜索或 MCP 服务器数据
- 审查数据结构，识别关键部分
- 定位财务报表（通常3-5年历史）
- 识别管理层预测（如有）
- 记录财年截止日期
- 立即标注任何数据质量问题

**步骤1.2：提取财务报表**
- 定位历史利润表数据
- 提取资产负债表快照（年末或季末）
- 查找现金流量表
- 提取管理层预测（如有）
- 记录所有页码以便追溯

**步骤1.3：提取运营指标**
- 识别与行业相关的非财务 KPI
- 获取单位经济数据
- 提取客户/门店/产能数据
- 记录增长指标和趋势

**步骤1.4：提取市场和行业数据**
- 竞争定位信息
- 市场规模和增长率
- 行业基准数据
- 同行比较信息

**步骤1.5：记录关键背景**
- 交易结构和逻辑
- 管理层团队背景
- 源材料中的投资亮点
- 风险因素和考量
- 任何数据缺口或不一致

### 阶段二：数据标准化与规范化

**步骤2.1：标准化会计列报**
- 确保所有年份的行项目名称一致
- 统一收入确认处理方式
- 识别并记录一次性费用
- 如需创建"调整后 EBITDA"调节表
- 记录任何会计政策变更

**步骤2.2：应用格式检测逻辑**
对每个数据点，根据完整上下文确定格式：
- 读取标签页名称、表格标题、列标题和行标签
- 应用核心规则（见上文）
- 不确定时，查阅原始源文件
- 默认使用更简洁的格式（少即是多）

**步骤2.3：识别标准化调整**
需记录的常见调整：
- 重组费用（如确实非经常性则加回）
- 股权激励（按行业标准加回）
- 收购相关费用（加回，注明金额）
- 法律和解或诉讼费用（评估复发风险）
- 资产出售或减值（从经营业绩中剔除）
- 关联方调整（标准化至市场费率）
注：源引用格式因数据源而异（文档用页码，网络用 URL，MCP 数据用服务器引用）

**步骤2.4：创建调整明细表**
对每项标准化调整：
- 记录调整内容及原因
- 引用来源（文档页码、URL 或数据源引用）
- 量化每年的金额影响
- 评估复发风险
- 展示从报告数据到调整后数据的计算过程

**步骤2.5：验证数据完整性**
- 确认小计使用公式正确加总
- 验证资产负债表平衡
- 检查现金流与资产负债表变动一致
- 交叉核对各标签页数据一致性
- 标注任何差异以待调查

### 阶段三：构建 Excel 工作簿

**关键提示：使用 xlsx 技能进行所有 Excel 文件操作。操作前请阅读 xlsx 技能文档。**

**步骤3.1：创建标准化标签页结构**
创建工作簿，包含以下标签页：
- 执行摘要
- 历史财务数据
- 资产负债表
- 现金流
- 运营指标
- 物业表现（如适用）
- 市场分析
- 投资亮点

**步骤3.2：按规范格式构建每个标签页**
系统性应用格式规则：
- 标题：粗体、左对齐、11pt 字体
- 财务数据：货币格式 ¥#,##0.0 表示百万
- 运营数据：数字格式 #,##0（无 ¥）
- 百分比：0.0% 格式
- 年份：文本格式防止千位分隔符
- 负数：使用带括号的会计格式
- 下划线：小计上方单下划线，合计下方双下划线

**步骤3.3：插入计算公式**
- 所有小计和合计必须基于公式
- 适当时链接资产负债表至利润表
- 链接现金流至利润表和资产负债表
- 创建跨标签页引用用于验证
- 避免硬编码任何计算值

### 错误：硬编码行偏移

**不要使用相对偏移 - 当表格结构变化时会出错：**

```python
# ❌ 错误 - 脆弱的偏移引用
formula = f"=B{row-15}/B{row-19}"  # row-15 是什么？row-19 是什么？

# ❌ 错误 - 魔术数字
formula = f"=B{current_row-10}*C{current_row-20}"
```

**为什么这会失败：**
- 增删行时静默出错
- 阅读代码时无法验证正确性
- 在交付的 Excel 文件中造成调试噩梦

**步骤3.4：应用专业展示**
- 在每个数据标签页冻结首行和首列
- 设置合适的列宽（通常12-15个字符）
- 所有数字数据右对齐

### 阶段四：情景构建（如包含预测）

**管理层情景：**
按源材料提供的管理层预测呈现：
- 提取所有管理层假设
- 记录增长率、利润率改善、资本需求
- 标注关键驱动因素和敏感性
- 标注任何需要怀疑的"曲棍球棒"拐点
- 标注为"管理层情景"并清晰标识

**基准情景（风险调整后）：**
基于公司特定风险因素对管理层预测进行保守调整：
- 对收入增长打折，反映执行风险和历史预测准确度
- 基于行业基准和运营杠杆适度调整利润率改善假设
- 如增长依赖资本支出则增加资本支出假设
- 如被低估则增加营运资金需求
- 如适用，基于整合复杂度延迟协同效应实现
- 记录所有调整的理由和支持分析

**下行情景（可选但推荐用于 LBO 分析）：**
基于行业周期性和公司脆弱性的压力测试情景：
- 模拟反映衰退风险或竞争压力的收入下降
- 假设压力下的利润率压缩（销量去杠杆、定价压力）
- 测试合规性和流动性
- 评估下行保护
- 记录被压力测试的关键风险

**情景文档要求：**
创建假设明细表，展示：
- 各情景的关键假设（收入增长、利润率、资本支出%）
- 每项调整的理由
- 关键变量的敏感性分析
- 历史预测准确度（如有）
- 与行业基准的比较

### 阶段五：质量控制与验证

**步骤5.1：数据准确性检查**
验证：
- 每个数字可追溯至来源（抽查样本，引用文档/URL/服务器）
- 所有计算基于公式（无硬编码值）
- 小计和合计数学正确
- 年份显示无千位分隔符（2024 而非 2,024）
- 无公式错误：#REF!, #VALUE!, #DIV/0!, #N/A

**步骤5.2：格式一致性检查**
核实：
- 财务数据格式带 ¥ 符号
- 运营数据格式不带 ¥ 符号
- 百分比显示为 %（15.0% 而非 0.15）
- 负数使用括号（仅限财务数据）
- 标题粗体且左对齐
- 数字右对齐
- 年份为文本格式

**步骤5.3：结构完整性检查**
确认：
- 所有必要标签页存在且顺序正确
- 执行摘要简洁（适合一页）
- 所有关键指标全面覆盖
- 从摘要到细节的逻辑流畅
- 各标签页适当的详细程度
- 无缺失数据或不完整部分

**步骤5.4：专业展示检查**
审查：
- 最少边框（仅用于结构）
- 统一缩进（子项2个空格）

## 标准化模式

### EBITDA 常见调整

**1. 重组费用**
- 如确实非经常性则加回（设施关闭、一次性遣散费）
- 如公司每年都在重组则不加回
- 记录具体性质和非经常性理由
- 示例："2023年重组：300万元设施关闭，源材料有记录，一次性事件"

**2. 股权激励**
- 行业标准：私募股权分析中加回
- 视为非现金经营费用
- 各期间保持一致
- 如异常高或包含一次性授予需注明

**3. 收购相关费用**
- 加回交易费用、整合成本
- 按类型记录具体金额
- 不加回持续的整合投资
- 引用每项调整的来源

**4. 法律和解与诉讼**
- 如确属孤立事件则加回
- 评估复发风险（一次和解 vs. 诉讼模式）
- 记录和解性质
- 考虑是否属于正常经营

**5. 资产出售或减值**
- 从经营 EBITDA 中剔除资产出售损益
- 如确实非经常性则剔除减值费用
- 记录出售/减值的资产及原因
- 如资产产生经营收入则调整收入

**6. 关联方调整**
- 标准化高于市场的关联方费用（租金、管理费）
- 调整至市场费率并提供支持文档
- 剔除通过公司报销的个人开支
- 记录市场费率比较

### 保守 vs. 激进标准化

**管理层情景：**
- 包含管理层提出的所有调整
- 接受公司对"非经常性"的定义
- 更激进的 EBITDA 调整
- 用于理解管理层观点

**基准情景（推荐用于投资决策）：**
- 仅限明确的非经常性项目
- 对经常性"一次性"费用进行更严格审查
- 排除推测性调整
- 更保守，对投资委员会更具说服力

## 行业特定适配

### 科技/SaaS
需捕获的关键指标：
- ARR（年度经常性收入）和 MRR
- 按群组分列的客户数量
- CAC（客户获取成本）和 LTV（生命周期价值）
- 流失率（总流失和净流失）
- 净收入留存率
- Rule of 40（增长率% + EBITDA 利润率%）
- Magic number（销售效率）

格式说明：ARR 为货币（¥），客户数为数字（无 ¥），比率为 %

### 制造/工业
需捕获的关键指标：
- 产能和产能利用率 %
- 按产品线分列的产量
- 存货周转率
- 按产品线分列的毛利率
- 订单积压

格式说明：单位、产能为数字（无 ¥），利用率为 %，收入/成本为货币

### 房地产/酒店
需捕获的关键指标：
- 物业/客房/面积
- 入住率 %
- ADR（平均每日房价）- 货币格式
- RevPAR（每可用房收入）- 货币格式
- NOI（净经营收入）- 货币格式
- Cap rates %
- FF&E 准备金

格式说明：客房/面积为数字，入住率为 %，ADR/RevPAR 为货币

### 医疗/服务
需捕获的关键指标：
- 门店/设施数量
- 医疗人员/员工数量
- 患者/就诊量（数量指标）
- 每次就诊收入 - 货币
- 支付方组合 %
- 同店增长率 %

格式说明：门店/就诊量为数字，每次就诊收入为货币，比率为 %
