# 交易跟踪器

## 工作流程

### 第一步：交易设置

为每个交易记录：
- **交易名称/代号**：Project [名称]
- **客户**：卖方或买方名称
- **交易类型**：卖方、买方、融资、重组
- **角色**：牵头顾问、联合顾问、公允意见
- **交易规模**：预期企业价值
- **阶段**：委托前 → 已签约 → 市场推广 → IOI → 尽调 → 最终报价 → 签约 → 交割
- **团队**：已分配的 MD、VP、Associate、Analyst
- **关键日期**：签约日期、CIM 分发、IOI 截止日、管理层会议、最终报价截止日、目标交割日

|-----------|------------|-------------|--------|-------|
| 委托函签署 | | | | |

| 买方名单审批 | | | | |
| Teaser 分发 | | | | |

| CIM 分发 | | | | |

状态：正常推进 / 存在风险 / 已延迟 / 已完成

### 第三步：待办事项

维护跨所有交易的待办事项列表：

| 待办事项 | 交易 | 负责人 | 截止日期 | 优先级 | 状态 |
|--------|------|-------|----------|----------|--------|
| | | | | P0/P1/P2 | 进行中/已完成/受阻 |

### 第四步：每周交易审查

为每周团队会议生成摘要：

**每个活跃交易：**
1. 一行状态更新
2. 本周关键进展
3. 近期里程碑（未来2周）
4. 阻碍因素或风险
5. 下周待办事项

**管线摘要：**
- 按阶段汇总活跃交易数量
- 风险交易（错过里程碑、流程停滞）
- 新委托/推介中的交易
- 本季度预期交割

### 第五步：输出

- Excel 工作簿，包含：
  - 管线总览（所有交易，一行一个）
  - 各交易里程碑跟踪标签页
  - 待办事项主清单
  - 每周审查摘要
- 可选：Markdown 格式摘要，用于邮件/Slack 分发

## 重要说明

- 每周至少更新一次跟踪器 —— 过时的跟踪器比没有跟踪器更糟
- 标注里程碑延迟的交易 —— 提前预警可避免意外
- 没有负责人和截止日期的待办事项不会被执行 —— 要具体
- 管线视图应显示交易阶段、规模和可能性 —— 有助于收入预测
- 记录买家/投资人反馈 —— 反馈中的规律可指导策略调整
- 将已关闭/已终止的交易单独归档 —— 保持活跃视图清晰
