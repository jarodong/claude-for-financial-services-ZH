## 视觉层级与布局

### 框体与章节布局

幻灯片布局因内容需求和模板设计而异。常见元素包括：
- 含标题和副标题的头部区域
- 带标签侧栏的内容框
- 结构化数据的表格
- 视觉数据展示的图表
- 幻灯片底部的脚注栏

具体布局应遵循所提供的模板。常见内容类型及其典型结构：
- **市场定义幻灯片**：带要点内容的标签框 + 评论区域
- **TAM/规模幻灯片**：指标高亮 + 数据表 + 关键要点
- **竞争分析**：比较表或矩阵
- **财务摘要**：图表配数据支撑表

### 对齐原则

**并排章节的垂直对齐：**

垂直堆叠的框体应保持一致的：
- 左边距位置
- 要点缩进
- 文字起始位置
- 框体宽度

水平相邻的框体应保持一致的：
- 顶部位置
- 高度（在内容允许的情况下）
- 内边距

## 文字格式

### 要点结构

避免非结构化的文字堆砌。将内容拆分为可扫描的要点。

**正确结构示例：**
```
✓  消费者移动端和网页语言学习应用
   （Duolingo、Babbel、Memrise、Busuu）
✓  B2B 企业语言培训平台
   （goFLUENT、Speexx、Learnship）
✓  在线辅导市场
   （italki、Preply、Cambly）
```

**错误结构示例（文字堆砌）：**
```
消费者移动/网页应用（Duolingo、Babbel、Memrise、Busuu）
B2B 企业平台（Speexx、Rosetta Stone Enterprise）
在线辅导市场（Preply、italki、Cambly）
```

### 要点符号指南

| 上下文 | 符号 | 用法 |
|---------|--------|-------|
| 包含/正面 | ✓（勾选） | 范围内的项目、存在的特性 |
| 排除/负面 | ×（叉号） | 范围外的项目、缺失的特性 |
| 中性列表 | •（圆点） | 一般列举、评论 |
| 编号序列 | 1. 2. 3. | 流程步骤、排名 |
| 子要点 | ‣ 或 – | 主要点下的二级要点 |

根据模板的现有惯例调整符号使用。

### 要点一致性

同一框体/章节内的所有要点应具有完全相同的格式：
- 整个框体使用相同的要点符号（除非有意区分）
- 所有主要点使用相同缩进级别
- 相同的要点大小
- 要点与文字之间的相同间距
- 同级要点文字使用相同字号

### 字号指南

以下为典型范围——根据模板规格调整：

| 元素 | 典型大小（pt） | 样式 |
|---------|-------------------|-------|
| 幻灯片标题 | 40-48 | 粗体 |
| 副标题/定义 | 18-22 | 粗体 |
| 章节标题 | 14-16 | 常规 |
| 正文/要点 | 12-14 | 常规 |
| 表头 | 10-12 | 粗体 |
| 表体 | 9-11 | 常规 |
| 脚注 | 8-9 | 斜体 |

### 文字密度指南

- 每个内容框最多 **6-7 个要点**（根据空间调整）
- 每个要点最多 **2 行**
- **括号示例**放在同行或缩进至下一行
- **避免孤词**——调整换行以避免单个词独占新行

## 表格创建

### 关键要求：使用实际表格对象

**表格必须是实际表格对象，而非带制表符间距的文本。**

带制表符的文本永远无法正确对齐，看起来不专业。始终创建正确的表格对象。

### 表格结构指南

1. **列对齐**：
   - 文本列：左对齐（标题和内容均左对齐）
   - 数字列：居中对齐或右对齐
   - 标题应与列内容对齐方式一致

2. **标题行**：
   - 粗体文字
   - 带阴影背景（使用模板的品牌色）
   - 对比色文字以提高可读性
   - 对齐方式与列内容一致

3. **交替行**（可选）：
   - 交替行使用浅色阴影提高可读性

4. **汇总/合计行**：
   - 粗体文字
   - 上方加粗边框（分隔线）
   - 不同的背景阴影

5. **表格宽度**：
   - 填满指定章节宽度
   - 避免表格悬浮在空白区域

### XML 实现模式参见 [`xml-reference.md`](xml-reference.md#table-implementation)

## 图表与图片处理

### 从 Excel 粘贴图表

从 Excel 粘贴图表时：

1. **仅粘贴图表**——不包含源数据表
2. **调整大小以填满指定区域**——图表不应显示为小缩略图
3. **保持宽高比**——不要拉伸变形
4. **验证可读性**——坐标轴标签、图例、数据标签必须清晰可读

### 从 Excel 粘贴表格

从 Excel 粘贴表格时：

1. **仅粘贴格式化后的表格**——排除任何源数据或计算公式
2. **调整大小以填满指定区域**——表格应占据其完整章节
3. **验证列宽**——调整以确保文字不被截断
4. **检查格式保留**——颜色、边框、字体可能需要调整

### 大小指南

**最小尺寸原则：**
- 图表：应占据指定区域的大部分空间
- 表格：完全填满指定章节宽度
- 图片：根据上下文适当大小，绝不能是缩略图大小

**过小视觉元素的标志（应避免）：**
- 图表仅占可用空间的一小部分
- 文字标签不可读
- 视觉元素周围大量空白区域
- 视觉元素显示为"缩略图"

### 正确的调整大小工作流程

1. 确定目标区域尺寸
2. 粘贴图表/表格
3. 立即调整大小以填满目标区域
4. 验证所有文字仍然可读
5. 如需要，调整内部元素（图例位置、坐标轴标签）

## 数据可视化

### 关键指标展示

展示关键指标（如 TAM、CAGR、预测值）时，考虑展示数值之间的关系，而非静态罗列：

- **视觉流向指示器**：使用形状（箭头、V 形、连接线）展示进展
- **大小层级**：主要指标使用较大字号，标签使用较小字号
- **空间排列**：将元素放置在能展示逻辑流向的位置

### 箭头和流向指示器

如果使用箭头或流向指示器：
- 使用 PowerPoint 形状对象，而非文字字符
- 不在最终演示文稿中使用文字箭头（→、⟹）
- 使用 PowerPoint 的形状工具或通过 XML 形状元素创建箭头

**XML 实现参见 [`xml-reference.md`](xml-reference.md#arrow-shapes)**

## 字体一致性

### 跨框体字体一致性

所有同一层级的文字框应使用完全相同的字号。

**应匹配的同级框体：**

| 框体类型 | 应与之匹配 |
|----------|-------------------|
| "包含细分"内容 | "排除细分"内容 |
| "定义"内容 | "范围理由"内容 |
| 左列要点 | 右列要点 |
| 所有标签框 | 彼此之间 |
| 所有章节标题 | 彼此之间 |

### 验证流程

1. 识别所有同一层级的文字框
2. 检查每个框体的字号
3. 如有差异，统一调整所有框体
4. 如内容能容纳，使用较大字号；否则统一使用较小字号

**例外**：子要点或次要文字可使用比主要点更小的字号，但必须在所有框体中保持一致。

## 模板适配

这些标准应根据所使用的具体模板进行调整：

1. **颜色**：使用模板的品牌色，而非指定特定颜色
2. **字体**：使用模板的字体系列
3. **间距**：匹配模板现有的间距惯例
4. **布局**：遵循模板的章节结构

无论模板如何，以下关键原则始终不变：
- 文字必须在背景上可读
- 表格必须是实际表格对象
- 内容应适当填满可用空间
- 并排元素的格式应一致
- 图表/图片应正确调整大小
