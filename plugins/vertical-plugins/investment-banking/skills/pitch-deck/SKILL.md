## 关键渲染限制

**LibreOffice 用于验证但不能准确渲染 PowerPoint 文件。** 它会破坏字体、渐变、形状位置、文字换行和部分表格格式。

**这意味着：** 在 LibreOffice 中通过视觉验证的幻灯片在 Microsoft PowerPoint 中可能仍有问题。验证循环可捕获结构问题（缺失内容、损坏的表格、保留的占位符格式）但**无法**捕获字体替换、细微对齐偏移或渐变问题。

**必要操作：** 交付输出时始终包含以下声明：
> "此文件使用 LibreOffice 验证。分发前请在 Microsoft PowerPoint 中审查，可能存在渲染差异。"

## 模板填充工作流程

复制并跟踪进度：

```
推介材料进度：
- [ ] 阶段一：提取和验证源数据
- [ ] 阶段二：将内容映射到模板部分
- [ ] 阶段三：按规范格式填充幻灯片
- [ ] 阶段四：验证 → 修复 → 重复直至清洁
- [ ] 阶段五：最终验证
```

### 阶段一：数据提取
1. 在任何修改之前**创建原始模板的备份** —— 复制为 `[文件名]_backup.pptx`。直接 XML 编辑或意外错误可能损坏文件。
2. 识别所有源材料（Excel、CSV、PDF 报告、Word 文档、数据库、网络来源）
3. 从每个源提取相关数据点
4. 根据原始源验证所有数字
5. 标准化单位和货币（将所有数字转换为模板使用的主要单位/货币）
6. 标注需要验证的计算 → 参见 [`calculation-standards.md`](reference/calculation-standards.md) 中的公式

### 阶段二：内容映射
1. **打开并视觉审查模板** —— 修改前了解其结构、样式和现有内容
2. 分析模板结构 —— 识别所有占位符区域和内容框
3. 将源数据映射到对应的模板部分 → 参见 [`slide-templates.md`](reference/slide-templates.md) 中的映射指南
4. 识别占位符指导框（任务创建者提供的彩色说明框）
5. 标注任何数据缺口或不匹配 → 参见 [`slide-templates.md`](reference/slide-templates.md#handling-data-template-mismatches) 中的解决方案

### 阶段三：模板填充
1. **删除或重新格式化占位符框** —— 彩色说明框展示要创建什么内容，而非如何格式化。删除它们并在其位置创建格式正确的内容。参见[关键反模式](#关键反模式绝不可为之)。
2. 用映射的内容填充每个部分（先关注内容）
3. **然后应用格式**以匹配模板样式 → 参见 [`formatting-standards.md`](reference/formatting-standards.md)
4. 将表格创建为实际表格对象（绝不使用竖线/制表符分隔的文本） → 参见 [`xml-reference.md`](reference/xml-reference.md#table-implementation)
5. 将箭头/形状创建为 PowerPoint 对象 → 参见 [`xml-reference.md`](reference/xml-reference.md#arrow-shapes)
6. 如任务文件中提供了公司 logo 则插入；如未提供，提醒用户："[LOGO NOT PROVIDED - please supply company logo]"

### 阶段四：验证 → 修复 → 重复

**这是一个反馈循环。重复直到所有检查通过或触发升级。**

```bash
# 转换为图片进行视觉验证
soffice --headless --convert-to pdf presentation.pptx
pdftoppm -jpeg -r 150 presentation.pdf slide
```

**验证清单（检查每张幻灯片图片）：**
- [ ] 文字在背景上可读？
- [ ] 表格是实际对象（列对齐，非竖线/制表符分隔的文本）？
- [ ] 图表/表格填满指定区域？
- [ ] 各部分内要点格式一致？
- [ ] 同级框的字体大小匹配？
- [ ] 无内容超出幻灯片边界？
- [ ] **无保留的占位符格式**（无大型彩色框内填入数据）？
- [ ] **无文本"表格"**（无用 `|` 或制表符创建的假列）？
- [ ] **跨幻灯片一致性**：相同指标/数字在所有出现的幻灯片中完全一致？

**修复循环协议：**

| 循环 | 操作 |
|-------|--------|
| 1 | 修复所有已识别问题，重新验证 |
| 2 | 修复剩余问题，重新验证 |
| 3 | 如问题持续，记录剩余问题并升级给用户 |

**3次循环后如仍有问题：**
1. 列出每个未解决的问题，标注幻灯片编号和描述
2. 说明已尝试的修复方法
3. 交付文件时附带明确声明："以下问题无法自动解决：[列表]。需要手动审查。"

**不要**无限循环。某些问题（字体渲染、复杂形状对齐）可能需要在 PowerPoint 中手动干预。

### 阶段五：最终验证

交付前运行[最终质量清单](#最终质量清单)。

### 幻灯片层级（典型）

这些是典型范围——根据模板规格调整：

| 层级 | 示例 | 典型大小 | 样式 |
|-------|----------|--------------|-------|
| 标题 | 幻灯片标题 | 40-48pt | 粗体 |
| 副标题 | 市场定义、幻灯片描述 | 18-22pt | 粗体 |
| 章节标题 | "关键预测"、"评论" | 14-16pt | 常规 |
| 区块标签 | "包含的细分"、"定义"侧栏 | 12-14pt | 常规 |
| 区块内容 | 要点、正文 | 11-14pt | 常规 |
| 表头 | 列标题 | 10-12pt | 粗体 |
| 表体 | 单元格内容 | 9-11pt | 常规 |
| 脚注 | 来源、注释 | 8-9pt | 斜体 |

### 字体一致性匹配

**同一层级**的框必须使用完全相同的字体大小：

| 同级 | 必须匹配 |
|------------|-----------------|
| "包含的细分" | "排除的细分" |
| "定义" | "范围理由" |
| 左列要点 | 右列要点 |
| 所有区块标签 | 彼此之间 |
| 所有章节标题 | 彼此之间 |

### 展示舍入

这些是**典型惯例** —— 根据数值大小和模板样式调整：

| 数值类型 | 典型舍入 | 示例 |
|------------|------------------|---------|
| 大型市场规模（100亿+） | 最近 10亿 | 18.5 → 190亿 |
| 较小市场规模（<100亿） | 最近 5亿 或 1亿 | 2.3 → 25亿 |
| 规模区间 | 匹配来源精度 | 14.9-22.1 → 150-220亿 |
| CAGR | 整数% 或 0.5% | 16.4% → 16% 或 16.5% |
| 市场份额 | 最近 5% 或匹配来源 | 21.4% → 20% |
| 倍数 | 1位小数 | 9.69 → 9.7x |

**原则：** 舍入不应实质性改变数值。对于较小的值，使用更精细的精度。

### 文字密度规则

- 每个内容框最多6-7个要点
- 每个要点最多2行
- 括号示例：同行或缩进至下一行
- 避免孤字（单个词独占新行）

### 对齐原则

**垂直堆叠的框**必须具有相同的：
- 左边距位置、要点缩进、文字起始位置、框宽度

**水平相邻的框**必须具有相同的：
- 顶部位置、高度（尽可能）、内部边距

### 跨幻灯片一致性

当相同数据出现在多张幻灯片时：
- 使用完全相同的数字、格式和术语
- 如在一张幻灯片更新了指标，需更新所有出现处
- 验证时交叉引用以捕获不匹配

## 强制要求

无论模板如何，以下要求不可协商：

| 要求 | 详情 |
|-------------|---------|
| **文字可读性** | 所有文字必须与背景有足够对比度。示例：深蓝、深绿、黑色背景上的白色/浅色文字；白色、浅灰、浅黄背景上的黑色/深色文字。 |
| **实际表格对象** | 表格数据必须是表格对象，非制表符分隔的文本。参见 [`xml-reference.md`](reference/xml-reference.md#table-implementation)。 |
| **正确的图表/表格大小** | 粘贴的视觉元素必须填满指定区域。参见 [`formatting-standards.md`](reference/formatting-standards.md#chart-and-image-handling)。 |
| **格式一致性** | 各部分内要点必须匹配（符号、大小、缩进）。同级框必须使用相同字体大小。 |
| **内容边界** | 所有内容必须保持在幻灯片边缘内。脚注框宽度：16:9 约32.5cm，4:3 约24cm。 |
| **无占位符格式** | 删除彩色说明框。正文：按模板使用浅色背景上的深色文字。 |

## 关键反模式：绝不可为之

当占位符格式被误认为输出格式时，会出现这些失败。识别这些模式至关重要。

### 反模式一：将数据填入占位符框

**发生了什么：** 模板有彩色说明框（黄色、橙色等），内含指导文字。模型用实际数据替换了指导文字，但**保留了彩色框**。

**为什么这是错误的：** 彩色框就是占位符。它告诉你那里放什么内容。输出应该有不同的格式 —— 通常是白色/浅色背景上的深色文字，或正确样式的形状。

**识别测试：** 如果你填充的幻灯片有大型彩色矩形内填满数据文本，你复制的是占位符格式而非替换它。

**关键区分 —— 两种"占位符"：**

| 类型 | 如何识别 | 应做处理 |
|------|-----------------|------------|
| **说明框** | 鲜艳颜色（黄色、橙色），含指导文字如"在此处插入 X"，彩色背景上的白色/浅色文字 | 删除整个形状，然后创建具有成品格式的新内容 |
| **布局占位符** | 幻灯片母版/布局的一部分，与模板主题匹配的中性颜色，"单击添加文字" | 保留形状，仅替换文字内容 |

如不确定：检查同一模板的空白幻灯片上是否存在该形状。布局占位符会持续存在；说明框是常规形状。

### 反模式二：文本"表格"

**发生了什么：** 模型使用分隔符（`|`、制表符、空格）创建类似表格的内容，而非实际表格对象。

**为什么这是错误的：** 这不是表格。列永远无法正确对齐，无法统一格式，看起来不专业。

**识别测试：** 如果你在输入 `|` 字符或依赖空格/制表符来创建列，你创建的是文本，不是表格。

**必须验证：** 创建任何表格后，验证它是实际表格对象。参见 [`xml-reference.md`](reference/xml-reference.md#critical-verify-tables-are-actual-table-objects) 中的验证方法。

### 反模式三：继承占位符对比度

**发生了什么：** 占位符使用彩色背景上的浅色文字（如黄色背景上的白色文字）。模型填充了数据但保留了此配色方案，导致输出难以阅读。

**为什么这是错误的：** 占位符颜色故意不同以示意"替换我"。成品幻灯片通常在正文区域使用浅色背景上的深色文字。

**识别测试：** 如果你填充的内容在正文区域（非标题）使用鲜艳彩色背景上的浅色/白色文字，你继承了占位符格式。

**正确方法：** 应用成品格式 —— 正文区域通常使用白色或浅色背景上的深色文字（#000000 或 #333333）。标题和强调区域可使用品牌色。

### 总结：占位符 vs. 成品

| 元素 | 占位符（输入） | 成品（输出） |
|---------|---------------------|---------------------|
| 说明框 | 彩色背景、指导文字 | 删除或重新格式化 |
| 数据区域 | "[在此处插入数据]"文字 | 带有简洁格式的实际数据 |
| 表格 | 表格应包含内容的描述 | 带行列的实际表格对象 |
| 正文 | 彩色背景上的浅色文字 | 浅色背景上的深色文字 |

**占位符告诉你创建什么，而非如何格式化。**

## 错误处理

**如 PDF/图片转换失败：**
1. 检查 LibreOffice 是否已安装：`which soffice`
2. 尝试替代方案：`libreoffice --headless --convert-to pdf presentation.pptx`
3. 如仍失败，手动在 PowerPoint/LibreOffice 中打开并导出

**如源数据存在不一致或冲突：**
1. **优先级**：优先使用任务文件中明确提供的数据
2. 如使用其他来源的数据（网络搜索、外部文档），提醒用户
3. 明确记录任何差异
4. 添加脚注解释数据来源选择

**如计算结果与源预测不匹配：**
1. 展示你的计算方法
2. 标注差异及可能原因（不同基准年、方法论）
3. 如差异重大，同时展示两个数值
4. 提醒用户解决

## 表格结构指南

创建表格时（必须是实际表格对象）：

**列对齐：**
- 文本列：左对齐（标题和内容）
- 数字列：右对齐或居中对齐（标题匹配内容）

**标题行：**
- 粗体文字
- 带阴影背景（模板品牌色）
- 白色或对比色文字

**共识/合计行：**
- 粗体文字
- 上方分隔线
- 不同的背景阴影

**宽度：** 完全填满指定部分宽度。

XML 实现参见 [`xml-reference.md`](reference/xml-reference.md#table-implementation)。

## 脚注格式

**格式：**
```
来源：[来源1]（年份），[来源2]（年份）。
注释：(1) [第一条注释]；(2) [第二条注释]。
```

**示例：**
```
来源：Grand View Research (2024), Mordor Intelligence (2024), Markets and Markets (2023)。
注释：(1) 不含硬件收入；(2) 包含 B2B 和 B2C 两个细分。
```

幻灯片正文中所有上标数字（¹, ², ³）必须在注释条目中有对应说明。
