# 差异根因追溯

给定单行差异（键值、GL 值、子账簿值、分类、可能原因），追溯至源头并生成根因说明。

## 追溯路径

1. **提取 GL 端**——通过 internal-gl MCP，获取产生该 GL 行的记账凭证或过账记录：凭证编号、过账日期、来源系统、批次编号、制单人。
2. **提取子账簿端**——通过子账簿 MCP，获取对应交易：交易编号、交易/结算日期、对手方、数据源、使用的汇率。
3. **对比属性**——逐一比对过账日期、汇率/日期、科目映射、数量方向、金额方向。存在差异的属性通常即为原因。

## 原因 → 说明

以单句格式撰写根因：**"⟨某端⟩ ⟨做了什么⟩ 因为 ⟨原因⟩"**，例如：

- "GL 在结算日（T+2）过账，而子账簿在交易日过账——时间性差异，将于 2026-05-07 消除。"
- "子账簿使用 WM/R 下午4点汇率；GL 使用 Wind 收盘汇率——基准金额上存在 12 bps 的汇率差异。"
- "证券 ABC123 在映射表中对应 GL 科目 11420，但子账簿传入的是 11410——映射差异，需提交至参考数据组处理。"
- "子账簿对该交易重复过账（交易编号 88412 和 88419 为重复记录）——重复过账，需抑制 88419。"

## 输出

对每条已追溯的差异，返回：

```json
{
  "key": "...",
  "root_cause": "单句说明，格式如上",
  "owner": "ops | reference-data | accounting | upstream-system",
  "expected_clear_date": "YYYY-MM-DD 或 null",
  "action": "monitor | adjust | raise-ticket | suppress"
}
```

只有调账人员才能编写调账分录——本技能仅负责诊断，不负责过账。
