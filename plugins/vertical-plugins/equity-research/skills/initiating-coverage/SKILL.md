# 首次覆盖

通过结构化的5个任务工作流创建机构级股票研究首次覆盖报告。每个任务必须单独执行并验证输入。

## 概述

本技能按照机构标准（摩根大通、高盛、摩根士丹利格式）产出全面的首次覆盖报告。任务逐个执行，每个任务在继续前验证前置条件。

**默认字体**：全文使用Times New Roman（除非用户另有指定）。

---

## 关键要求：一次一个任务

**本技能仅支持单任务模式运行。**

### 如果用户要求完整流程

当用户要求：
- "为[公司]创建首次覆盖报告"
- "为[公司]撰写首次覆盖报告"
- "为[公司]完成整个股票研究流程"
- "为[公司]完成所有5个任务"
- 任何暗示运行多个任务或整个工作流的请求

**必须回复：**

1. **询问要执行哪个具体任务：**
   ```
   我可以帮您为[公司]创建股票研究首次覆盖报告。
   这需要5个单独的任务，需逐一完成：

   1. 公司研究 - 研究业务、管理层、行业
   2. 财务建模 - 建立预测模型
   3. 估值分析 - DCF和可比公司
   4. 图表生成 - 创建25-35张图表
   5. 报告组装 - 编制最终报告

   您想从哪个任务开始？
   ```

2. **当用户明确要求所有任务一起完成时：**
   ```
   我理解您希望完成整个首次覆盖报告流程。
   目前，本技能支持一次执行一个任务，这样可以在每个阶段
   进行更好的质量控制和审查。

   我们正在开发无缝的端到端工作流，使这个过程更加自动化，
   但目前需要分别完成每个任务。

   您想从任务1（公司研究）开始吗？
   ```

3. **永远不要自动假设从哪个任务开始**——始终请用户确认。

4. **永远不要按顺序执行多个任务**——完成一个任务、交付输出，然后等待用户的下一个请求。

### 任务执行规则

- 每次用户请求只执行一个任务
- 开始任务前始终验证前置条件
- 交付任务输出并确认完成
- 等待用户明确请求下一个任务
- 永远不要自动将多个任务串联
- 永远不要假设用户要继续下一个任务
- 永远不要在未验证所需输入存在的情况下执行任务3-5

### 交付物政策：不走捷径

**只交付指定的输出。不要创建额外文档。**

每个任务指定确切交付物。不要创建：
- "完成摘要"
- "执行摘要"
- "快速参考指南"
- "下一步文档"
- "任务完成报告"
- 任何其他未明确指定的"有帮助"文档

**原因**：这些额外内容浪费上下文，且不属于专业工作流。

**应该交付的内容**：
- 任务1：研究文档（.md）——仅此而已
- 任务2：财务模型（.xlsx）——仅此而已
- 任务3：估值分析（.md）+ 添加到任务2文件的Excel标签页——仅此而已
- 任务4：图表压缩文件（.zip）——仅此而已
- 任务5：最终报告（.docx）——仅此而已

**如果交付物未在上面列出，就不要创建。**

         询问从哪个任务开始（见上面模板）

```
User: "I want to complete all 5 tasks for 宁德时代"
Response: 不要将任务串联
         解释一次一个的限制（见上面模板）
         询问是否要从任务1开始
```

### 正确使用示例

**执行单个任务：**
```
"Use initiating-coverage skill, Task 1 for 宁德时代"
"Do Task 2 of initiating-coverage for 宁德时代"
"Run Task 3 for 宁德时代 using the initiating-coverage skill"
```

**完成完整报告（需要5次独立请求）：**
```
Request 1: "Do Task 1 for 宁德时代" → 完成 → 交付输出
Request 2: "Do Task 2 for 宁德时代" → 完成 → 交付输出
Request 3: "Do Task 3 for 宁德时代" → 完成 → 交付输出
Request 4: "Do Task 4 for 宁德时代" → 完成 → 交付输出
Request 5: "Do Task 5 for 宁德时代" → 完成 → 交付输出
```

### 任务执行顺序

对完整的首次覆盖报告，任务必须按以下顺序在独立的用户请求中执行：

```
Request 1: Task 1 - 公司研究（独立）
           ↓ [用户审查输出并请求下一个任务]
Request 2: Task 2 - 财务建模（独立）
           ↓ [用户审查输出并请求下一个任务]
Request 3: Task 3 - 估值分析（需要任务2输出）
           ↓ [用户审查输出并请求下一个任务]
Request 4: Task 4 - 图表生成（需要任务2和3输出）
           ↓ [用户审查输出并请求下一个任务]
Request 5: Task 5 - 报告组装（需要所有前面任务的输出）
```

**注意**：任务1和2可以按任意顺序运行。任务3-5有严格依赖关系，必须在继续前验证输入。

## 任务1：公司研究

**目的**：研究公司的业务、管理层、竞争地位、行业和风险。

**前置条件**：无（完全独立）
- 公司名称或股票代码

**流程**：
1. 验证已提供公司名称/股票代码
2. 从references/task1-company-research.md加载详细说明
3. 执行定性研究工作流
4. 交付研究文档

**输出**：公司研究文档（6,000-8,000字）
- 公司概况与历史
- 管理层简介（每位300-400字 × 3-4位高管）
- 产品与服务分析
- 行业概览
- 竞争分析（5-10个竞争对手）
- TAM规模
- 风险评估（8-12个风险）

**文件名**：`[Company]_Research_Document_[Date].md`

**只交付这1个文件。不要完成摘要，不要额外文档。**

**不要走捷径：**
- 写完整的6,000-8,000字（非摘要）
- 为所有3-4位高管完成300-400字的简介
- 彻底分析所有5-10个竞争对手
- 覆盖4类风险中的所有8-12个风险
- 不要为节省时间而缩减章节
- 不要跳过任何必需章节

**继续前的验证**：此任务无需验证。

## 任务2：财务建模

**目的**：提取历史财务数据并构建全面的Excel财务模型，包含预测和情景。

**前置条件**：开始前需验证
- **必须**：访问公司财务数据
  - 上市公司：巨潮资讯网最新年报
  - 非上市公司：财务报表或可用预估
  - 或：用户提供的已提取历史财务数据
- **可选**：公司研究（任务1）用于业务背景

**输入验证**：
```
开始前——选择方式：

方式A：提取财务数据（最常见）
- [ ] 可访问年报或财务报表？
- [ ] 准备好提取3-5年数据？

方式B：用户提供已提取的财务数据
- [ ] 已收到历史财务文件？
- [ ] 包含利润表、现金流量表、资产负债表（3-5年）？

可选：
- [ ] 公司研究（任务1）已完成，可提供背景？
```

**流程**：
1. 验证可访问财务数据
2. 从references/task2-financial-modeling.md加载详细说明
3. **步骤1**：提取历史财务数据（如需要）
4. **步骤2+**：构建包含6个核心标签页的预测模型
5. 交付Excel模型

**输出**：Excel财务模型（.xlsx）
- 6个核心标签页：
  1. **收入模型** - 产品细分（20-30行）+ 地域细分（15-20行）
  2. **利润表** - 完整损益表含40-50行项目，历史（3-5年）+ 预测（5年）
  3. **现金流量表** - 经营/投资/筹资活动，历史 + 预测
  4. **资产负债表** - 资产/负债/权益，历史 + 预测
  5. **情景** - 乐观/基准/悲观比较表
  6. **DCF输入** - 为任务3估值准备

**文件名**：`[Company]_Financial_Model_[Date].xlsx`

**只交付这1个文件。不要完成摘要，不要额外文档。**

**不要走捷径：**
- 如果提取财务数据：从3张财务报表提取所有行项目（3-5年）
- 完整构建所有6个预测标签页，含完整详情
- 创建详细的收入模型含20-30个产品行和15-20个地域行
- 构建完整的利润表含40-50行项目（非缩减版）
- 包含完整的现金流量表和资产负债表及所有行项目
- 完成所有三种情景（乐观/基准/悲观），采用不同参数
- 不要创建简化/缩减版本
- 不要跳过6个核心标签页中的任何一个
- 如需要，不要跳过历史财务数据提取

**继续任务3前的验证**：
- 历史财务数据已提取（如需要）或已提供
- Excel文件已创建且可打开
- 模型包含所有6个核心标签页（收入模型、利润表、现金流量、资产负债表、情景、DCF输入）
- 已纳入历史数据（3-5年）
- 预测完成（5年前瞻）
- 情景完成（乐观/基准/悲观）
