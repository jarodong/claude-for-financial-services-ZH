# 论点跟踪器

## 工作流程

### 第一步：定义或加载论点

如果创建新论点：
- **公司**：名称和股票代码
- **持仓**：多头或空头
- **论点陈述**：1-2句核心论点（例如，"做多ACME——定价权带来的利润率扩张 + 组合转向软件带来的经营杠杆"）
- **关键支柱**：3-5个支撑论据
- **主要风险**：3-5个可能使论点失效的风险
- **催化剂**：即将发生的可能证实/证伪论点的事件（财报、产品发布、监管决策）
- **目标价/估值**：如果论点实现，值多少
- **止损触发点**：什么情况会让你退出

如果更新现有论点，向用户确认新的数据点或发展。

### 第二步：更新日志

对每个新的数据点或发展：

- **日期**：发生时间
- **数据点**：什么变化了（财报超预期、管理层离任、竞争对手动作等）
- **论点影响**：这加强、削弱还是中性化了某个支柱？
- **行动**：不变/加仓/减仓/退出
- **更新确信度**：高/中/低

### 第三步：论点计分卡

维护持续更新的计分卡：

| 支柱 | 原始预期 | 当前状态 | 趋势 |
|------|---------|---------|------|
| 收入增长 >20% | 按计划 | Q3为22% | 稳定 |
| 利润率扩张 | 落后 | 利润率同比持平 | 值得关注 |
| 新产品发布 | 待定 | 延迟至Q2 | 观察 |

### 第四步：催化剂日历

跟踪即将发生的催化剂：

| 日期 | 事件 | 预期影响 | 备注 |
|------|------|---------|------|
| | | | |

### 第五步：输出

论点摘要适用于：
- 晨会讨论
- 投资组合回顾
- 风险委员会汇报

格式：简洁的markdown或Word文档，包含计分卡、近期更新和当前确信度水平。

## 重要提示

- 论点应该是可证伪的——如果没有任何东西能推翻它，那就不是论点
- 跟踪反证和证实证据一样严格
- 至少每季度审查一次论点，即使没有重大事件发生
- 如果用户管理多个持仓，主动提出进行全面的投资组合论点审查
- 以结构化格式存储论点数据，以便跨会话引用
