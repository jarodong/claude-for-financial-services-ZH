# 可比公司分析

## ⚠️ 关键：数据源优先级（先读此处）

**始终遵循此数据源层级：**

1. **首先：检查 MCP 数据源**——如果 Wind MCP / Choice MCP、Wind / iFinD MCP 可用，专门使用它们获取财务和交易信息
2. **如果上述 MCP 数据源可用，不要使用网页搜索**
3. **仅在 MCP 不可用时：** 才使用 Wind 终端 / iFinD 终端、巨潮资讯网文件或其他机构来源
4. **绝不用网页搜索作为主要数据源**——缺乏机构级分析所需的准确性、审计跟踪和可靠性

**原因：** MCP 来源提供经过验证的机构级数据和正确引用。网页搜索结果可能过时、不准确或不可靠。

---

## 概述

此 Skill 教 Claude 构建机构级可比公司分析，结合运营指标、估值倍数和统计基准。输出是结构化的 Excel/电子表格，通过同行比较支持明智的投资决策。

**参考材料与情境化：**

`examples/comps_example.xlsx` 中提供了可比公司分析示例。使用此或此 Skill 目录中的其他示例文件时，要智能使用：

**应该用示例做：**
- 理解结构层级（节如何流动）
- 把握预期的严谨程度（统计深度、文档标准）
- 学习原则（清晰标题、透明公式、审计跟踪）

**不要用示例做：**
- 精确复制格式或指标
- 不考虑上下文地复制布局
- 不管受众地应用相同视觉风格

**始终先问自己：**
1. **"您有偏好格式还是我应该适应模板风格？"**
2. **"受众是谁？"**（投决会、董事会演示、快速参考、详细备忘录）
3. **"关键问题是什么？"**（估值、增长分析、竞争定位、效率）
4. **"上下文是什么？"**（M&A 评估、投资决策、行业对标、业绩审查）

**根据具体情况调整：**
- **行业上下文**：科技巨头需要的指标与新兴 SaaS 初创公司不同
- **行业特有需求**：尽早添加相关指标（如云 ARR、企业客户、开发者生态）
- **公司熟悉度**：知名公司可能需要更少背景，更多关注变化分析
- **决策类型**：M&A 需要的侧重点与持续组合监控不同

**核心原则：** 使用模板原则（清晰结构、统计严谨、透明公式），但根据上下文调整执行。目标是机构级质量分析，而非机构级外观模板。

用户提供的示例和明确偏好始终优先于默认值。

## 核心理念
**"先建正确结构，然后让数据讲故事。"**

从强制思考什么重要的标题开始，输入干净数据，构建透明公式，让统计自动呈现。好的 Comps 应该让没构建它的人也能立即读懂。

---

## ⚠️ 关键：公式优先于硬编码 + 逐步验证

**环境——Office JS vs Python：**
- **如果在 Excel 中运行（Office 加载项 / Office JS）：** 直接使用 Office JS（`Excel.run(async (context) => {...})`）。通过 `range.formulas = [["=E7/C7"]]` 写入公式，而非 `range.values`。无需单独重算步骤——Excel 原生处理。使用 `range.format.*` 设置颜色/字体。
- **如果生成独立 .xlsx 文件：** 使用 Python/openpyxl。写入 `cell.value = "=E7/C7"`（公式字符串）。
- 两种方式相同原则——只需转换 API 调用。
- **Office JS 合并单元格陷阱：** 不要先调用 `.merge()` 再在合并范围上设置 `.values`（会抛出 `InvalidArgument`——范围仍报告合并前尺寸）。改为先在左上角单元格写入值，然后合并+格式化整个范围：
  ```js
  ws.getRange("A1").values = [["TECHNOLOGY — COMPARABLE COMPANY ANALYSIS"]];
  const hdr = ws.getRange("A1:H1");
  hdr.merge();
  hdr.format.fill.color = "#1F4E79";
  hdr.format.font.color = "#FFFFFF";
  hdr.format.font.bold = true;
  ```

**公式，而非硬编码：**
- 每个派生值（利润率、倍数、统计量）必须是引用输入单元格的 Excel 公式——绝不是粘贴的预计算数字
- 使用 Python/openpyxl 构建工作表时：写入 `cell.value = "=E7/C7"`（公式字符串），而非 `cell.value = 0.687`（计算结果）
- 唯一应硬编码的值是原始输入数据（收入、EBITDA、股价等）——且每个都要有带来源的单元格注释
- 原因：模型必须在输入变化时自动更新。硬编码利润率是等待发生的静默错误。

**逐步与用户验证：**
- 设置结构后 → 向用户展示标题布局再填数据
- 输入原始数据后 → 向用户展示输入块，确认来源/期间再建公式
- 构建运营指标公式后 → 向用户展示计算出的利润率并合理性检查，再转入估值
- 构建估值倍数后 → 向用户展示倍数确认合理，再添加统计量
- 不要端到端构建整个工作表再展示——通过确认每节尽早发现错误

## 第 1 节：文档结构与设置

### 标题块（第 1-3 行）
```
第 1 行：[分析标题] - 可比公司分析
第 2 行：[公司列表含代码] • [公司 1 (代码1)] • [公司 2 (代码2)] • [公司 3 (代码3)]
第 3 行：截至 [期间] | 除每股金额和比率外，所有数字单位为 [人民币百万元/亿元]
```

**原因：** 立即建立上下文。任何打开此文件的人都知道在看什么、何时创建、如何解读数字。

### 视觉约定标准（可选——用户偏好和上传模板始终优先）

**重要：** 这些仅是建议默认值。始终优先考虑：
1. 用户明确的格式偏好
2. 上传模板文件的格式
3. 公司/团队风格指南
4. 这些默认值（仅在无其他指导时）

**建议字体与排版：**
- **字体族**：Times New Roman（专业、可读、行业标准）
- **字号**：数据单元格 11pt，标题 12pt
- **粗体文本**：节标题、公司名称、统计标签

**默认颜色与阴影——专业蓝灰色调（越少越好）：**
- **保持克制**——仅蓝色和灰色。不要引入绿色、橙色、红色或多种强调色。干净的 Comps 表总共使用 3-4 种颜色。
- **节标题**（如"运营统计与财务指标"）：
  - 深蓝背景（`#1F4E79` 或 `#17365D` 海军蓝）
  - 白色粗体文本
  - 跨所有列的整行阴影
- **列标题**（如"公司"、"收入"、"利润率"）：
  - 浅蓝背景（`#D9E1F2` 或类似淡蓝）
  - 黑色粗体文本
  - 居中对齐
- **数据行**：
  - 公司数据白色背景
  - 公式黑色文本；硬编码输入蓝色文本
- **统计行**（最大值、75 分位等）：
  - 浅灰背景（`#F2F2F2`）
  - 黑色文本，左对齐标签
- **全部色板**：深蓝 + 浅蓝 + 浅灰 + 白色。除非用户模板另有规定，不使用其他颜色。

**建议格式约定：**
- **小数精度**：
  - 百分比：1 位小数（12.3%）
  - 倍数：1 位小数（13.5x）
  - 人民币金额：无小数，千位分隔符（69,632）
  - 利润率显示为百分比：1 位小数（68.7%）
- **边框**：无边框（干净、极简外观）
- **对齐**：所有指标居中对齐，呈现干净统一外观
- **单元格尺寸**：所有列宽统一/均等，所有行高一致（创建干净、专业的网格）

**注意：** 如果用户提供模板文件或指定不同格式，使用该格式。

## 第 2 节：运营统计与财务指标

### 核心列（从这些开始）
1. **公司**——名称格式一致
2. **收入**——规模指标（可以是 LTM、季度或年度，取决于上下文）
3. **收入增长**——同比百分比变化
4. **毛利**——收入减销售成本
5. **毛利率**——毛利/收入（基础盈利能力）
6. **EBITDA**——息税折旧摊销前利润
7. **EBITDA 利润率**——EBITDA/收入（运营效率）

### 可选补充（根据行业/目的选择）
- **季度 vs LTM**——如果季节性重要，两者都包含
- **自由现金流**——对资本密集型或 SaaS 企业
- **FCF 利润率**——FCF/收入（现金生成效率）
- **净利润**——对成熟、盈利公司
- **营业利润**——对 D&A 变化大的企业
- **资本支出指标**——对重资产行业
- **Rule of 40**——专门对 SaaS（增长% + 利润率%）
- **FCF 转化率**——对盈利质量分析（高级）

### 公式示例（以第 7 行为例）
```excel
// 核心比率——始终计算
毛利率 (F7): =E7/C7
EBITDA 利润率 (H7): =G7/C7

// 可选比率——相关时包含
FCF 利润率: =[FCF]/[收入]
净利润率: =[净利润]/[收入]
Rule of 40: =[增长%]+[FCF 利润率%]
```

**黄金法则：** 每个比率应是 [某指标] / [收入] 或 [某指标] / [本表某指标]。保持简单。

### 统计块（公司数据之后）

**关键：为所有可比指标（比率、利润率、增长率、倍数）添加统计公式。**

```
[留一空行用于视觉分隔]
- 最大值: =MAX(B7:B9)
- 75 分位: =QUARTILE(B7:B9,3)
- 中位数: =MEDIAN(B7:B9)
- 25 分位: =QUARTILE(B7:B9,1)
- 最小值: =MIN(B7:B9)
```

**需要统计的列（可比指标）：**
- 收入增长%、毛利率%、EBITDA 利润率%、EPS
- EV/收入、EV/EBITDA、P/E、股息收益率%、Beta

**不需要统计的列（规模指标）：**
- 收入、EBITDA、净利润（绝对规模因公司规模而异）
- 市值、企业价值（不同规模公司间不可比）

**注意：** 在公司数据和统计行之间留一空行用于视觉分隔。不要添加"行业统计"或"估值统计"标题行。

**原因：** 四分位显示分布，而非仅平均值。75 分位倍数告诉你"溢价"公司的交易水平。

## 第 3 节：估值倍数与投资指标

### 核心估值列（从这些开始）
1. **公司**——与运营节相同顺序
2. **市值**——当前市场估值
3. **企业价值**——市值 ± 净债务/现金
4. **EV/收入**——市场为每元销售支付多少
5. **EV/EBITDA**——市场为每元盈利支付多少
6. **P/E 倍数**——价格相对于净利润

### 可选估值指标（根据上下文选择）
- **FCF 收益率**——FCF/市值（对现金导向分析）
- **PEG 比率**——P/E/增长率（对成长型公司）
- **市净率**——市场价值 vs 账面价值（对重资产企业）
- **ROE/ROA**——回报指标（对盈利能力比较）
- **收入/EBITDA CAGR**——历史增长率（对趋势分析）
- **资产周转率**——收入/资产（对运营效率）
- **债务/权益**——杠杆（对资本结构分析）

**关键原则：** 包含 3-5 个对行业重要的核心倍数。不要因为可以就包含所有可能指标。

### 公式示例
```excel
// 核心倍数——始终包含
EV/收入: =[企业价值]/[LTM 收入]
EV/EBITDA: =[企业价值]/[LTM EBITDA]
P/E 倍数: =[市值]/[净利润]

// 可选倍数——数据可用时包含
FCF 收益率: =[LTM FCF]/[市值]
PEG 比率: =[P/E]/[增长率%]
```

### 交叉引用规则
**关键：** 估值倍数必须引用运营指标节。绝不两次输入相同原始数据。如果收入在 C7，则 EV/收入公式应引用 C7。

### 统计块
与运营节相同结构：每个指标的最大值、75 分位、中位数、25 分位、最小值。在公司数据和统计之间留一空行。不要添加"估值统计"标题行。

## 第 4 节：注释与方法论文档

### 必需组件

**数据来源与质量：**
- 数据来自哪里？（Wind MCP / Choice MCP、Wind / iFinD MCP、Wind 终端 / iFinD 终端、上市公司公告文件）
- 覆盖什么期间？（2024 年 Q4、审计数据）
- 如何验证？（与年度报告/季度报告交叉核对）
- 注意：优先使用 MCP 数据源（Wind MCP、Choice MCP、Wind / iFinD）以获得更好的准确性和可追溯性

**关键定义：**
- EBITDA 计算方法（毛利 + D&A，或营业利润 + D&A）
- 自由现金流公式（经营现金流 - 资本支出）
- 特殊指标解释（Rule of 40、FCF 转化率）
- 时间区间定义（LTM、CAGR 计算期间）

**估值方法论：**
- 企业价值如何计算？（市值 + 净债务）
- 使用了什么增长率？（历史 CAGR、前瞻性估计）
- 做了什么调整？（剔除一次性项目、正常化利润率）

**分析框架：**
- 投资论点是什么？（云/SaaS 效率）
- 什么指标最重要？（现金生成、资本效率）
- 读者应如何解读统计量？（四分位提供上下文）

## 第 5 节：选择正确指标（决策框架）

### 从"我在回答什么问题？"开始

**"哪家公司被低估？"**
→ 关注：EV/收入、EV/EBITDA、P/E、市值
→ 跳过：运营细节、增长指标

**"哪家公司最高效？"**
→ 关注：毛利率、EBITDA 利润率、FCF 利润率、资产周转率
→ 跳过：规模指标、绝对人民币金额

**"哪家公司增长最快？"**
→ 关注：收入增长%、EBITDA CAGR、用户/客户增长
→ 跳过：利润率指标、杠杆比率

**"哪家是最佳现金生成者？"**
→ 关注：FCF、FCF 利润率、FCF 转化率、资本支出强度
→ 跳过：EBITDA、P/E 比率

### 行业特有指标选择

**软件/SaaS：**
必备：收入增长、毛利率、Rule of 40
可选：ARR、净人民币留存率、CAC 回收期
跳过：资产周转率、存货指标

**制造/工业：**
必备：EBITDA 利润率、资产周转率、资本支出/收入
可选：ROA、存货周转率、订单积压
跳过：Rule of 40、SaaS 指标

**金融服务：**
必备：ROE、ROA、效率比率、P/E
可选：净息差、贷款损失准备
跳过：毛利率、EBITDA（对银行无意义）

**零售/电商：**
必备：收入增长、毛利率、存货周转率
可选：同店销售、客户获取成本
跳过：重研发或资本支出指标

### "5-10 法则"

**5 个运营指标**——收入、增长、2-3 个利润率/效率指标
**5 个估值指标**——市值、EV、3 个倍数
**= 共 10 列**——足以讲故事，不会多到迷失主线

如果超过 15 个指标，可能包含了噪音。果断删减。

## 第 6 节：最佳实践与质量检查

### 开始前
1. **定义同行组**——公司必须真正可比（类似商业模式、规模、地域）
2. **选择正确期间**——LTM 平滑季节性；季度显示趋势
3. **预先标准化单位**——百万 vs 十亿的决定影响一切
4. **映射数据来源**——知道每个数字来自哪里

### 构建中
1. **先输入所有原始数据**——在写公式前完成蓝色文本
2. **为所有硬编码输入添加单元格注释**——右键单元格 → 插入注释 → 记录来源或假设

   **对有来源的数据，精确引用出处：**
   - 示例："Wind 终端 - 贵州茅台 (600519.SH)，访问于 2024-10-02"
   - 示例："2024 年年度报告，第 42 页，行项目'Total Revenue'"
   - 示例："Wind / iFinD 一致预期，截至 2024-10-02"
   - **尽可能包含超链接**：右键单元格 → 链接 → 粘贴上市公司公告文件、数据源或报告的 URL

   **对假设，解释推理：**
   - 示例："基于同行中位数假设 15% EBITDA 利润率，公司未披露"
   - 示例："估计企业价值为市值 + 5000万元 净债务（来自 Q3 资产负债表，Q4 尚未可用）"
   - 示例："前瞻性 P/E 基于分析师一致预期 EPS 3.45元（12 位分析师估计的平均值）"

   **原因：** 实现审计跟踪、数据验证、假设透明和未来更新
3. **逐行构建公式**——在继续前测试每个计算
4. **对标题使用绝对引用**——$C$6 锁定标题行
5. **格式一致**——百分比显示为百分比，而非小数
6. **添加条件格式**——自动高亮异常值

### 合理性检查
- **利润率测试**：毛利率 > EBITDA 利润率 > 净利润率（按定义始终成立）
- **倍数合理性**：
  - EV/收入：通常 0.5-20x（因行业差异大）
  - EV/EBITDA：通常 8-25x（跨行业相当一致）
  - P/E：通常 10-50x（取决于增长率）
- **增长-倍数相关性**：更高增长通常意味着更高倍数
- **规模-效率权衡**：大公司通常有更好的利润率（规模优势）

### 常见错误
❌ 在公式中混淆市值和企业价值
❌ 分子分母使用不同时间区间（LTM vs 季度）
❌ 在公式中硬编码数字而非单元格引用
❌ **硬编码输入没有单元格注释引用来源或解释假设**
❌ 有上市公司公告文件或数据源时缺少超链接
❌ 包含太多指标而无明确目的
❌ 包含不可比公司（不同商业模式）
❌ 使用过时数据而未披露
❌ 错误计算百分比平均值（应为中位数）

## 第 6 节：高级功能

### 动态标题
对显示计算的列，使用清晰单位标签：
```
收入增长 (YoY) % | EBITDA 利润率 | FCF 利润率 | Rule of 40
```

### 四分位分析优势
不仅有均值/中位数，四分位显示：
- **75 分位** = "溢价"公司在此交易
- **中位数** = 典型市场估值
- **25 分位** = "折价"区域

这帮助回答："我们的目标公司相对于同行交易偏贵还是偏便宜？"

### 行业特有修改

**软件/SaaS：**
- 添加：ARR、净人民币留存率、CAC 回收期
- 强调：Rule of 40、FCF 利润率、毛利率 >70%

**医疗保健：**
- 添加：研发/收入、管线价值、监管状态
- 强调：EBITDA 利润率、增长率、报销风险

**工业：**

**消费：**

## 第 7 节：工作流与实操技巧

### 逐步流程
1. **设置结构**（30 分钟）
   - 创建所有标题
   - 格式化单元格（蓝色输入，黑色公式）
   - 锁定单位和日期引用

2. **收集数据**（60-90 分钟）
   - 从主要来源提取（Wind MCP、Choice MCP、Wind / iFinD MCP 如可用；否则 Wind 终端 / iFinD 终端、上市公司公告）
   - 用蓝色输入所有原始数字
   - 在注释节记录来源

3. **构建公式**（30 分钟）
   - 从简单比率开始（利润率）
   - 进阶到倍数（EV/收入）
   - 添加交叉检查（利润率合理吗？）

4. **添加统计**（15 分钟）
   - 为所有列复制公式结构
   - 验证范围正确（B7:B9，而非 B7:B10）
   - 检查四分位逻辑

5. **质量控制**（30 分钟）
   - 运行合理性检查
   - 验证公式引用
   - 检查 #DIV/0! 或 #REF! 错误
   - 与已知基准比较

6. **文档化**（15 分钟）
   - 完成注释节
   - 添加数据来源
   - 定义方法论
   - 日期戳分析

### 专业技巧
- **保存模板**：构建一次，永久复用
- **颜色编码异常值**：对 >2 个标准差的值使用条件格式
- **链接到源文件**：超链接到 Wind 终端 / iFinD 终端截图或上市公司公告文件
- **版本控制**：保存为 "Comps_v1_2024-12-15" 带清晰日期
- **协作审查**：让别人检查你的公式

### Excel 格式清单（可选——适应用户偏好）
- [ ] 字体设为用户偏好风格（默认：Times New Roman，数据 11pt，标题 12pt）

## 第 10 节：红旗与警告信号

### 数据质量问题
🚩 时间区间不一致（混合季度和年度）
🚩 缺失数据未解释
🚩 数据来源间差异显著（>10% 差异）

### 估值红旗
🚩 负 EBITDA 公司用 EBITDA 倍数估值（改用收入倍数）
🚩 P/E 比率 >100x 而无超增长故事
🚩 利润率对行业而言不合理

### 可比性问题
🚩 不同财年结束（导致时间差异）
🚩 混合纯业务和集团
🚩 商业模式显著不同的公司被标为"可比"

**有疑问时，排除该公司。** 3 个完美可比好过 6 个可疑的。

## 第 11 节：公式参考指南

### 基本 Excel 公式
```excel
// 统计函数
=AVERAGE(range)          // 简单均值
=MEDIAN(range)           // 中位数
=QUARTILE(range, 1)      // 25 分位
=QUARTILE(range, 3)      // 75 分位
=MAX(range)              // 最大值
=MIN(range)              // 最小值
=STDEV.P(range)          // 标准差

// 财务计算
=B7/C7                   // 简单比率（利润率）
=SUM(B7:B9)/3            // 多家公司平均值
=IF(B7>0, C7/B7, "N/A")  // 条件计算
=IFERROR(C7/D7, 0)       // 处理除零

// 跨表引用
='Sheet1'!B7             // 引用另一工作表
=VLOOKUP(A7, Table1, 2)  // 从数据表查找
=INDEX(MATCH())          // 高级查找

// 格式化
=TEXT(B7, "0.0%")        // 格式化为百分比
=TEXT(C7, "#,##0")       // 千位分隔符
```

### 常见比率公式
```excel
毛利率 = 毛利 / 收入
EBITDA 利润率 = EBITDA / 收入
FCF 利润率 = 自由现金流 / 收入
FCF 转化率 = FCF / 经营现金流
ROE = 净利润 / 股东权益
ROA = 净利润 / 总资产
资产周转率 = 收入 / 总资产
债务/权益 = 总债务 / 股东权益
```

## 关键原则总结

1. **结构驱动洞察**——正确标题强制正确思考
2. **少即是多**——5-10 个重要指标胜过 20 个不重要的
3. **为问题选择指标**——估值分析 ≠ 效率分析
4. **统计显示模式**——中位数/四分位比均值揭示更多
5. **透明胜过复杂**——人人理解的简单公式
6. **可比性为王**——宁可排除也不要强行比较
7. **记录选择**——在注释节解释哪些指标及原因
