# 三表财务模型模板完成

完成并填充整合财务模型模板，在利润表、资产负债表和现金流量表之间建立正确的链接。

## ⚠️ 关键原则——填充任何模板前必读

**环境——Office JS vs Python：**
- **如果在 Excel 中运行（Office 加载项 / Office JS）：** 直接使用 Office JS。通过 `range.formulas = [["=D14*(1+Assumptions!$B$5)"]]` 写入公式——对派生单元格绝不使用 `range.values`。无需单独重算；Excel 原生计算。使用 `context.workbook.worksheets.getItem(...)` 导航工作表。
- **如果生成独立 .xlsx 文件：** 使用 Python/openpyxl。写入 `ws["D15"] = "=D14*(1+Assumptions!$B$5)"`，然后在交付前运行 `recalc.py`。
- **Office JS 合并单元格陷阱：** 不要先调用 `.merge()` 再在合并范围上设置 `.values`——会抛出 `InvalidArgument`，因为范围仍报告合并前的尺寸。改为先在左上角单元格写入值，然后合并+格式化整个范围：`ws.getRange("A1").values = [["INCOME STATEMENT"]]; const h = ws.getRange("A1:G1"); h.merge(); h.format.fill.color = "#1F4E79";`
- 以下所有原则在两种环境中完全相同。

**公式优先于硬编码（不可妥协）：**
- 每个预测单元格、滚动结转、链接和小计必须是 Excel 公式——绝不是预计算值
- 使用 Python/openpyxl 时：写入公式字符串（`ws["D15"] = "=D14*(1+Assumptions!$B$5)"`），而非计算结果（`ws["D15"] = 12500`）
- 唯一应包含硬编码数字的单元格是：(1) 历史实际值，(2) 假设工作表中的假设驱动因素
- 如果你发现自己在 Python 中计算值并写入结果——停下。改为写公式。
- 原因：模型必须在情景切换或假设变化时灵活响应。硬编码会无声破坏每个下游完整性检查。

**逐步与用户验证：**
1. **映射模板后** → 向用户展示已识别的工作表/节，在触碰任何单元格前确认
2. **填充历史数据后** → 向用户展示历史块，确认值/期间与源数据匹配
3. **构建 IS 预测后** → 运行小计检查，向用户展示预测 IS，在转入 BS 前确认
4. **构建 BS 后** → 向用户展示每个期间的平衡检查（资产 = 负债+权益），在转入 CF 前确认
5. **构建 CF 后** → 向用户展示现金衔接（CF 期末现金 = BS 现金），在最终确认前确认
6. **不要端到端填充整个模型再完整展示**——在每张报表处停下展示工作，尽早发现错误

## 格式——专业蓝灰色调（默认，除非模板/用户另行指定）

**保持颜色极简。** 仅使用蓝色和灰色作为单元格填充。不要引入绿色、黄色、橙色或多种强调色——干净的模型讲究克制。

| 元素 | 填充 | 字体 |
|------|------|------|
| 节标题（IS / BS / CF 标题） | 深蓝 `#1F4E79` | 白色粗体 |
| 列标题（FY2024A、FY2025E 等） | 浅蓝 `#D9E1F2` | 黑色粗体 |
| 输入单元格（历史值、假设驱动因素） | 浅灰 `#F2F2F2` 或白色 | 蓝色 `#0000FF` |
| 公式单元格 | 白色 | 黑色 |
| 跨表链接 | 白色 | 绿色 `#008000` |
| 检查行 / 关键合计 | 中蓝 `#BDD7EE` | 黑色粗体 |

**3 种蓝色 + 1 种灰色 + 白色。** 如果模板有自己的配色方案，遵循模板。

字体颜色表示单元格*是*什么（输入/公式/链接）。填充颜色表示你*在*哪里（标题/数据/检查）。

## 模型结构

### 识别模板工作表组织

模板的工作表命名约定和组织方式各不相同。填充前，检查所有工作表以了解模板结构。以下是常见工作表名称及其典型内容：

| 常见工作表名称 | 内容 |
|----------------|------|
| IS、P&L、Income Statement | 利润表 |
| BS、Balance Sheet | 资产负债表 |
| CF、CFS、Cash Flow | 现金流量表 |
| WC、Working Capital | 营运资本计划 |
| DA、D&A、Depreciation、PP&E | 折旧摊销计划 |
| Debt、Debt Schedule | 债务计划 |
| NOL、Tax、DTA | 净营业亏损计划 |
| Assumptions、Inputs、Drivers | 驱动假设和输入 |
| Checks、Audit、Validation | 错误检查仪表板 |

**模板审查清单**
- 识别模板中存在哪些工作表（并非所有模板都包含每个计划）
- 注意上面未列出的模板特有工作表
- 了解工作表依赖关系（如哪些计划馈入主报表）
- 在每个工作表上定位输入单元格 vs 公式单元格

### 理解模板结构

填充模板前，熟悉其现有布局，确保数据输入到正确位置且公式保持完整。

**识别行结构**
- 在每个工作表顶部找到模型标题
- 识别节标题及其视觉分隔
- 找到表示 百万元、%、x 等的单位行
- 注意区分实际值 vs 预测值期间的列标题
- 确认期间标签（如 FY2024A、FY2025E）
- 识别输入单元格 vs 公式单元格（通常通过字体颜色区分）

**识别列结构**
- 确认最左侧的行项目标签
- 验证历史年份在预测年份之前
- 注意分隔历史与预测期间的视觉边框
- 检查所有工作表的列顺序一致

**使用命名范围**
模板通常对关键输入和输出使用命名范围。输入数据前：
- 检查模板中的现有命名范围（Excel 中公式 → 名称管理器）
- 常见命名范围包括：收入增长率、成本百分比、关键输出（净利润、EBITDA、总债务、现金）、情景选择器单元格
- 确保输入在馈入这些命名范围的单元格中

### 预测期
- 模板通常从最后历史年份向前预测 5 年
- 验证历史（A）vs 预测（E）列清晰分隔
- 确认列使用财年表示法（如 FY2024A、FY2025E）

## 利润率分析

**注意：以下利润率分析仅在用户提示或模板明确要求时执行。如无提示，跳过此节。**

在利润表（IS）工作表上计算和显示盈利能力利润率，以跟踪运营效率并支持同行比较。

### 核心利润率

| 利润率 | 公式 | 衡量内容 |
|--------|------|----------|
| 毛利率 | 毛利 / 收入 | 定价能力、生产效率 |
| EBITDA 利润率 | EBITDA / 收入 | 核心运营盈利能力 |
| EBIT 利润率 | EBIT / 收入 | 折旧摊销后运营盈利能力 |
| 净利润率 | 净利润 / 收入 | 底线盈利能力 |

### 含利润率的利润表布局

在每个利润行项目下方直接显示利润率百分比：
- 毛利率 % 在毛利下方
- EBIT 利润率 % 在 EBIT 下方
- EBITDA 利润率 % 在 EBITDA 下方
- 净利润率 % 在净利润下方

## 信用指标

**注意：以下信用分析仅在用户提示或模板明确要求时执行。如无提示，跳过此节。**

在资产负债表（BS）工作表上计算和显示信用/杠杆指标，以评估财务健康、债务能力和契约合规性。

### 核心信用指标

| 指标 | 公式 | 衡量内容 |
|------|------|----------|
| 总债务 / EBITDA | 总债务 / LTM EBITDA | 杠杆倍数 |
| 净债务 / EBITDA | (总债务 - 现金) / LTM EBITDA | 扣除现金后的杠杆 |
| 利息覆盖率 | EBITDA / 利息费用 | 偿债能力 |
| 债务 / 总资本 | 总债务 / (总债务 + 权益) | 资本结构 |
| 债务 / 权益 | 总债务 / 总权益 | 财务杠杆 |
| 流动比率 | 流动资产 / 流动负债 | 短期流动性 |
| 速动比率 | (流动资产 - 存货) / 流动负债 | 即时流动性 |

### 信用指标层级检查

验证乐观情景显示最强信用状况：
- 杠杆：乐观 < 基准 < 悲观（越低越好）
- 覆盖率：乐观 > 基准 > 悲观（越高越好）
- 流动性：乐观 > 基准 > 悲观（越高越好）

### 契约合规跟踪

如果已知债务契约，添加显式合规检查，将实际指标与契约阈值比较。

## 情景分析（基准 / 乐观 / 悲观）

使用假设工作表中的情景切换（下拉菜单），配合 CHOOSE 或 INDEX/MATCH 公式。

| 情景 | 描述 |
|------|------|
| 基准情景 | 管理层指引或一致预期 |
| 乐观情景 | 超指引增长、利润率扩张 |
| 悲观情景 | 低于趋势增长、利润率压缩 |

**关键敏感性驱动因素**：收入增长、毛利率、SG&A%、DSO/DIO/DPO、资本支出%、利率、税率。

**情景审计检查**：切换影响所有报表、所有情景下 BS 平衡、现金衔接、层级成立（乐观 > 基准 > 悲观 对 NI、EBITDA、FCF、利润率）。

## 完成模型模板

本节提供完成任何三表财务模型模板的通用指导，同时保留现有公式并确保数据完整性。

### 第 1 步：分析模板结构

输入任何数据前，彻底审查模板以了解其架构：

**识别输入 vs 公式单元格**
- 寻找区分输入单元格和公式单元格的视觉线索（字体颜色、单元格阴影）
- 常见约定：蓝色字体 = 输入，黑色字体 = 公式，绿色字体 = 链接到其他工作表
- 使用 Excel 的追踪引用/依赖项（公式 → 追踪引用）了解单元格关系
- 检查可能控制关键输入的命名范围（公式 → 名称管理器）

**映射模板流程**
- 识别哪些工作表馈入其他工作表（如假设 → IS → BS → CF）
- 注意任何辅助计划及其与主报表的链接
- 在填充前记录模板的特有行项目和结构

### 第 2 步：不破坏公式地填入数据

**数据输入黄金规则**

| 规则 | 描述 |
|------|------|
| 仅编辑输入单元格 | 除非有意替换公式，否则绝不覆盖包含公式的单元格 |
| 保留单元格引用 | 复制数据时，使用粘贴值（Ctrl+Shift+V）避免用源格式覆盖公式 |
| 匹配模板单位 | 输入数据前验证模板使用千、百万还是实际值 |
| 遵守符号约定 | 遵循模板现有符号约定（如费用为正或负） |
| 检查循环引用 | 如果模板使用迭代计算，确保启用了迭代计算 |

**安全数据输入流程**
1. 识别指定输入的精确单元格（通常高亮或标注）
2. 先输入历史数据，然后验证公式对这些期间计算正确
3. 输入馈入预测计算的假设驱动因素
4. 检查计算输出确认公式按预期工作
5. 如果必须修改公式单元格，在更改前记录原始公式

**处理预建公式**
- 如果公式引用你尚未填充的单元格，在所有输入完成前预期临时错误（#REF!、#DIV/0!）
- 公式产生意外结果时，追踪引用以识别缺失或不正确的输入
- 在检查所有工作表的公式依赖关系前，绝不删除行/列

### 第 3 步：验证公式

**公式完整性检查**

在依赖模板输出前，验证公式正确运行：

| 检查类型 | 方法 |
|----------|------|
| 追踪引用 | 选择公式单元格 → 公式 → 追踪引用，验证其引用正确输入 |
| 追踪依赖项 | 验证关键输入流向预期输出单元格 |
| 评估公式 | 使用公式 → 评估公式逐步执行复杂计算 |
| 检查硬编码 | 预测公式应引用假设，不包含硬编码值 |
| 用已知值测试 | 输入简单测试值验证公式产生预期结果 |
| 跨表一致性 | 确保相同公式逻辑适用于所有预测期间 |

**常见公式问题**
- 混合绝对/相对引用导致跨期间复制时结果错误
- 链接到外部文件或已删除范围的断链（#REF! 错误）
- 收入增长前早期期间的除零错误（#DIV/0! 错误）
- 循环引用警告（可能是利息计算的有意设计）
- 预测列间公式不一致（使用 Ctrl+\ 查找差异）

**验证跨表链接**
- 确认出现在多个工作表上的值是链接的（非重复的）
- 验证计划合计链接到主报表的对应行项目
- 检查期间标签在所有工作表上对齐

### 第 4 步：按工作表质量检查

填充模板后在每个工作表上执行这些验证检查：

**利润表（IS）质量检查**
- 历史期间的收入数字与源数据匹配
- 所有费用行项目加总到报告合计
- 小计（毛利、EBIT、EBT、净利润）计算正确
- 税务计算逻辑适当（正确处理亏损）
- 预测驱动因素引用假设工作表（无硬编码）
- 逐期变化方向合理

**资产负债表（BS）质量检查**
- 每个期间资产 = 负债 + 权益（首要检查）
- 现金余额与现金流量表期末现金匹配
- 营运资本账户链接到辅助计划（如适用）
- 留存收益正确滚动结转：前期 RE + 净利润 - 股息 +/- 调整 = 期末 RE
- 债务余额链接到债务计划（如适用）
- 所有资产负债表项目有适当符号（资产为正，多数负债为正）

**现金流量表（CF）质量检查**
- CFO 顶部的净利润与利润表净利润匹配
- 非现金加回（D&A、SBC 等）链接到其源计划/报表
- 营运资本变化有正确符号（资产增加 = 现金使用 = 负）
- 资本支出链接到 PP&E 计划或固定资产滚动结转
- 融资活动链接到 BS 上债务和权益账户的变化
- 期末现金与资产负债表现金匹配
- 期初现金等于上期期末现金

**辅助计划质量检查**
- 期初余额等于上期期末余额
- 滚动结转逻辑完整（期初 + 增加 - 减少 = 期末）
- 计划合计链接到主报表行项目
- 计算中使用的假设与假设工作表匹配

### 第 5 步：跨报表完整性检查

验证单个工作表后，确认三张报表正确整合：

| 检查 | 公式 | 预期结果 |
|------|------|----------|
| 资产负债表平衡 | 资产 - 负债 - 权益 | = 0 |
| 现金衔接 | CF 期末现金 - BS 现金 | = 0 |
| 净利润链接 | IS 净利润 - CF 起始净利润 | = 0 |
| 留存收益 | 前期 RE + NI - 股息 - BS 期末 RE | = 0（按需调整 SBC/其他项目） |

### 第 6 步：最终审查

在认为模型完成前：
- 切换所有情景（如适用）验证每个情景下检查通过
- 检查所有 #REF!、#DIV/0!、#VALUE! 和 #NAME? 错误并解决或记录
- 确认所有输入单元格已填充（搜索占位符值）
- 验证所有工作表单位一致
- 在做任何额外修改前保存干净版本

## 模型验证与审计

本节汇总已完成模板的所有验证检查和审计程序。

### 核心链接（必须始终成立）

参见 [references/formulas.md](references/formulas.md) 了解所有公式详情。

| 检查 | 公式 | 预期结果 |
|------|------|----------|
| 资产负债表平衡 | 资产 - 负债 - 权益 | = 0 |
| 现金衔接 | CF 期末现金 - BS 现金 | = 0 |
| 月度 vs 年度现金 | 期末现金（月度） - 期末现金（年度） | = 0 |
| 净利润链接 | IS 净利润 - CF 起始净利润 | = 0 |
| 留存收益 | 前期 RE + NI + SBC - 股息 - BS 期末 RE | = 0 |
| 权益融资 | Δ普通股/APIC（BS） - 权益发行（CFF） | = 0 |
| 第 0 年权益 | 权益募集（第 0 年） - 期初权益资本（第 1 年） | = 0 |

### 符号约定参考

| 报表 | 项目 | 符号约定 |
|------|------|----------|
| CFO | D&A、SBC | 正（加回） |
| CFO | ΔAR（增加） | 负（现金使用） |
| CFO | ΔAP（增加） | 正（现金来源） |
| CFI | 资本支出 | 负 |
| CFF | 债务发行 | 正 |
| CFF | 债务偿还 | 负 |
| CFF | 股息 | 负 |

### 循环引用处理

利息费用产生循环：利息 → 净利润 → 现金 → 债务余额 → 利息

在 Excel 中启用迭代计算：文件 → 选项 → 公式 → 启用迭代计算。设置最大迭代次数 100，最大变化 0.001。在假设工作表中添加断路器切换。

### 检查类别

**第 1 节：货币一致性**
- 货币已识别并记录在假设中
- 所有工作表使用一致的货币符号和单位
- 单位行与模型货币匹配

**第 2 节：资产负债表完整性**
- 资产 = 负债 + 权益（每个期间）
- 公式：资产 - 负债 - 权益（必须 = 0）

**第 3 节：现金流量完整性**
- 现金衔接 BS（CF 期末现金 = BS 现金）
- 月度 vs 年度现金：期末现金（月度） = 期末现金（年度）
- NI 衔接 IS（CF 净利润 = IS 净利润）
- D&A 衔接计划
- SBC 衔接 IS
- ΔAR、Δ存货、ΔAP 衔接 WC 计划
- 资本支出衔接 DA 计划

**第 4 节：留存收益**
- RE 滚动结转检查：前期 RE + NI + SBC - 股息 = 期末 RE
- 显示组件明细用于调试

**第 5 节：营运资本**
- AR、存货、AP 衔接 BS
- DSO、DIO、DPO 合理性检查（超出正常范围时标记）

**第 6 节：债务计划**
- 总债务衔接 BS（流动 + 长期债务）
- 利息计算衔接 IS

**第 6b 节：权益融资**
- 权益发行收益衔接 BS 普通股/APIC 增加
- 权益带来的现金增加 = 权益账户增加（必须平衡）
- 权益衔接：Δ普通股/APIC（BS） = 权益发行（CFF）（必须 = 0）
- 第 0 年权益衔接：权益募集（第 0 年） = 期初权益资本（第 1 年）

**第 6c 节：NOL 计划**
- 期初 NOL（第 1 年 / 成立时） = 0（新企业从零 NOL 开始）
- NOL 仅在 EBT < 0 时增加（亏损必须实现才能产生 NOL）
- DTA 衔接 BS（NOL 计划 DTA = BS 递延所得税资产）
- NOL 使用 ≤ EBT 的 80%（2017 年后联邦限制）
- NOL 余额非负（不能使用超过可用量）
- NOL 仅在 EBT < 0 时产生
- 应税收入 ≤ 0 时税务费用 = 0

**第 7 节：情景层级**
- 绝对指标：乐观 > 基准 > 悲观（NI、EBITDA、FCF）
- 利润率：乐观 > 基准 > 悲观（GM%、EBITDA%、NI%）
- 信用指标：乐观 < 基准 < 悲观 对杠杆（反向）

**第 8 节：公式完整性**
- COGS、S&M、G&A、R&D、SBC 由收入百分比驱动（无硬编码）
- 预测年份间公式一致
- 无 #REF!、#DIV/0!、#VALUE! 错误

**第 9 节：信用指标阈值**
- 根据契约阈值将指标标记为绿/黄/红
- 任何红旗汇总

### 主检查公式

将所有节状态聚合为单一主检查：
- 如果所有节通过 → "✓ 所有检查通过"
- 如果任何节失败 → "✗ 检测到错误——请查看下方"

### 快速调试工作流

当主状态显示错误时：
1. 滚动找到红色高亮节
2. 识别哪个检查类别有失败
3. 导航到源工作表调查
4. 修复根本问题
5. 返回检查工作表验证解决
