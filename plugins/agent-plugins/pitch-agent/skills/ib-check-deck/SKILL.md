# IB Deck 检查器

对演示文稿进行四个维度的全面质检。读取每张幻灯片，然后报告发现。

## 环境检查

此 Skill 在 PowerPoint 加载项和聊天中均可工作。开始前识别你所在的环境：

- **加载项** —— 从实时打开的 Deck 读取。
- **聊天** —— 从上传的 `.pptx` 文件读取。

这是只读报告——不编辑——所以两种环境工作流相同。

## 工作流

### 读取 Deck

从每张幻灯片提取文本，跟踪每行来自哪张幻灯片。你需要幻灯片级归属用于每个发现（"5亿元 出现在第 3 和第 8 页，但第 15 页显示 4.85亿元"）。30 页的 Deck 太多无法可靠地保持在工作记忆中——将提取的文本写入文件，以便数字检查脚本处理。

脚本期望带有幻灯片标记的类 Markdown 输入。格式为：

```
## Slide 1
[第 1 页文本内容]

## Slide 2
[第 2 页文本内容]
```

### 1. 数字一致性

对你收集的内容运行提取脚本：

```bash
python scripts/extract_numbers.py /tmp/deck_content.md --check
```

它会规范化单位（5亿元 vs 500百万 vs 500,000,000 → 同一数字），分类值（revenue、EBITDA、倍数、利润率），并在同一指标类别在不同幻灯片上显示冲突值时标记。这是最可能捕捉到人类第五次通读时遗漏的内容。

除了脚本标记的内容，还要验证：
- 计算正确（加总求和、百分比加总、增长率与端点匹配）
- 单位风格一致——Deck 应选择百万元或千万元之一并坚持
- 时间区间对齐——FY vs LTM vs 季度，明确标注

### 2. 数据与叙述对齐

将声明映射到应该支撑它的数据。这是 Deck 静默出错的地方——有人编辑了第 7 页的图表，忘了第 4 页的叙述。

- 趋势声明（"margins declining"）——图表实际朝那个方向走吗？
- 市场地位声明（"#1 player"）——收入和份额数据支撑吗？
- 合理性——"#1 in a 10000亿元 market" 但收入 20亿元 是 0.2% 份额；那不是 #1

### 3. 语言打磨

IB Deck 有特定语域。扫描任何破坏它的东西：口语化表达（"pretty good"、"a lot of"）、缩写、感叹号、没有数字的模糊量词、同一概念的不一致术语。

参见 `references/ib-terminology.md` 了解替换模式。

### 4. 视觉和格式质检

对每张幻灯片运行标准视觉验证检查。你要找的是：缺失的图表来源引用、缺失的轴标签、排版不一致、数字格式漂移（同一 Deck 中 1,000 vs 1K）、日期格式漂移、脚注和免责声明缺失。

视觉验证捕捉文本提取中不显示的重叠、溢出和对比度问题。不要跳过——没有来源引用的图表在文本转储中看起来和有正确来源的一样。

## 输出

使用 `references/report-format.md` 作为结构。按严重程度分类：

- **Critical** —— 数字不匹配、事实错误、数据与叙述矛盾。这些阻止客户交付。
- **Important** —— 语言、缺失来源、术语漂移。应该修复。
- **Minor** —— 字体大小、间距、日期格式。打磨。

以 Critical 开头。如果没有，明确说明——"未发现数字不一致"是一个发现，而不是没有发现。
