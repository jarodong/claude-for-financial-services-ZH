# Deck 刷新

跨 Deck 更新数字。Deck 是格式的真相来源；你只改变数值。

## 环境检查

此 Skill 在 PowerPoint 加载项和聊天中均可工作。开始前识别你所在的环境——编辑机制不同，意图相同：

- **加载项** —— Deck 实时打开；直接编辑文本运行、表格单元格和图表数据。
- **聊天** —— Deck 是上传的文件；通过用新值重新生成受影响的幻灯片并写回文件来编辑。

无论哪种方式：尽可能小的改动，现有格式保持不变。

这是一个四阶段流程，第三阶段是审批门。在用户看到计划之前不要编辑。

## 阶段 1 —— 获取数据

使用 `ask_user_question` 了解新数字的来源：

- **粘贴映射** —— 用户输入或粘贴 "revenue 4.85亿元 → 5.12亿元, EBITDA 1.2亿元 → 1.35亿元"。最清晰的情况。
- **上传 Excel** —— 旧/新列，或用户想从中提取的新输出工作表。读取它，在信任之前确认哪列是哪列。
- **只有新值** —— "Q4 revenue was 5.12亿元, margins were 22%"。你来确定每个值替换什么。可行，但在触碰前确认映射——把 "5.12亿元" 映射到收入但用户指的是毛利润，是一个静默灾难。

还要问**衍生数字**：如果收入变了，用户想重新计算增长率和占比，还是保持不变？Deck 中某处可能有 "+15% YoY" 现在已过时。是否触碰这些是用户应该做的判断，不是你。

## 阶段 2 —— 读取一切，找到一切

读取每张幻灯片。对于每个旧值，找到每个实例——包括看起来不一样的：

| 变体 | 示例 |
|------|------|
| 规模 | `4.85亿元`、`4.85亿元`、`485,000,000元` |
| 精度 | `4.85亿元`、`4.85亿元`、`~4.85亿元` |
| 单位风格 | `4.85亿元`、`4.85亿元`、`4.85亿元`、`485百万元` |
| 嵌入式 | "revenue grew to 4.85亿元"、"a 4.85亿元 business"、轴标签 |

一个 Deck 在第 3 页说 `4.85亿元`，第 8 页图表轴上说 `485`，第 15 页脚注说 `4.85亿元`，这是同一个数字的三个实例。查找替换漏掉两个。你不应该漏。

**数字藏在哪里：**
- 文本框（明显）
- 表格单元格
- 图表数据标签和轴标签
- 图表源数据——驱动柱形的数字，不只是上面的标签
- 脚注、来源行、小字
- 演讲者备注（如果用户关心）

建立列表：对于每个旧值，它出现的每个位置、出现的确切文本、以及它将变成什么。这个列表就是计划。

## 阶段 3 —— 展示计划，获得批准

**这是对某人花时间制作的 Deck 的破坏性操作。** 在编辑任何东西前展示完整的变更列表。格式化为可扫描：

```
4.85亿元 → 5.12亿元 (Revenue)
  第 3 页  — 标题框："Revenue grew to 4.85亿元"
  第 8 页  — 图表轴标签："485"
  第 15 页 — 脚注："4.85亿元 in FY24 revenue"

1.2亿元 → 1.35亿元 (Adj. EBITDA)
  第 3 页  — 表格单元格
  第 11 页 — 正文："1.2亿元 of Adj. EBITDA"

标记 — 可能是衍生的，不在你的映射中：
  第 3 页  — "+15% YoY"（增长率——如果基准年没变，过时了？）
  第 7 页  — "12% market share"（是从 4.85亿元 / 市场规模计算的吗？）
```

标记部分很重要。你不只是执行查找替换——你在捕捉用户在晚上 11 点会遗漏的二阶效应。如果映射说 `4.85亿元 → 5.12亿元` 且第 3 页旁边还有 `+15% YoY`，那个增长率现在可能错了。标记它；不要静默修复，不要静默留下。

使用 `ask_user_question` 获得批准：按所示执行、执行但跳过标记项、或让他们先修改映射。

## 阶段 4 —— 执行，保留，报告

对于每个变更，完成它的最小编辑。如何发生取决于环境：

- **加载项** —— 在活动 Deck 中直接编辑特定运行、单元格或图表系列。
- **聊天** —— 用新值重新生成受影响的幻灯片，保留其他每个元素完全不变，写回文件。

无论哪种方式，标准相同：

- **形状中的文本** —— 改变值，保留字体/大小/颜色/粗体状态不变。如果 `4.85亿元` 是句子中的 14pt 海军蓝粗体，`5.12亿元` 就是同一句子中的 14pt 海军蓝粗体。
- **表格单元格** —— 改变单元格，保留表格不变。
- **图表数据** —— 更新底层系列值，使柱形/线条实际移动。只编辑标签而不改数据会让图表撒谎。

不要重新格式化任何不需要触碰的东西。Deck 的现有风格按定义是正确的；你是外科医生，不是装修工。

最后一次编辑后，报告实际发生了什么：

```
更新了 8 张幻灯片中的 11 个值。

已变更：
  [阶段 3 的列表，现在用过去时]

仍标记 — 未变更：
  第 3 页 — "+15% YoY"（衍生的；单独确认）
  第 7 页 — "12% market share"
```

对每张编辑过的幻灯片运行标准视觉验证检查。变长的数字（`4.85亿元` → `12.05亿元`）可能溢出文本框或推动表格列宽。在用户之前发现它。

## 你不要做的事

- **不重建幻灯片** —— 如果幻灯片的叙述在新数字下不再合理（"margins compressed" 但利润率上升了），标记它，不重写。
- **除非要求不重新计算** —— 衍生数字由用户决定。阶段 1 的问题覆盖了这点。
- **不碰格式** —— 如果 Deck 用 `亿元` 且用户的映射说 `百万元`，匹配 Deck，不匹配映射。值变；风格不变。
