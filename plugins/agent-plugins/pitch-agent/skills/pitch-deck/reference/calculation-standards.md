## 关键验证公式

### CAGR 预测

**公式：**
```
Future Value = Present Value × (1 + CAGR)^n
```

**变量：**
- Present Value：当前/基准年市场规模
- CAGR：复合年增长率（小数形式，如 16.4% = 0.164）
- n：基准年至目标年之间的年数

**验证示例：**
```
来源声称：221亿元（2024年）按 16.4% CAGR = 550亿元（2030年）

验证：22.1 × (1.164)^6 = 22.1 × 2.488 = 55.0 ✓
```

**计算 n（年数）：** 计算基准年和目标年之间的年数。示例：2024→2030 = 6年，2025→2030 = 5年。

### 估值倍数

**EV/收入：**
```
EV/收入倍数 = 企业价值 ÷ 收入
隐含 EV = 收入 × 倍数
```

**EV/EBITDA：**
```
EV/EBITDA 倍数 = 企业价值 ÷ EBITDA
隐含 EV = EBITDA × 倍数
```

**验证示例：**
```
来源声称：4.36亿元 交易，收入倍数 9.7x，收入 4500万元

验证：436 ÷ 45 = 9.69 ≈ 9.7x ✓
```

### 市场份额

**公式：**
```
市场份额 = (细分市场规模 ÷ 总市场规模) × 100
```

**验证示例：**
```
来源声称：在线细分（180亿元）占总市场（650亿元）的 28%

验证：18 ÷ 65 = 0.277 = 27.7% ≈ 28% ✓
```

### 增长率

**同比增长：**
```
同比增长 = (当年 - 上年) ÷ 上年 × 100
```

**由端点计算 CAGR：**
```
CAGR = (终值 ÷ 初值)^(1/n) - 1
```

## 共识方法论

当源数据包含多个估算值时，验证共识计算：

### 规模共识（区间）

**方法：** 全部来源的完整最小-最大区间

**示例：**
```
来源：149亿元、183亿元、211亿元、212亿元、221亿元
共识：150-220亿元（四舍五入至最近 10亿元）
```

### CAGR 共识（中心聚类）

**方法：** 排除异常值（最高和最低），使用中心聚类区间

**示例：**
```
来源：10.6%、16.4%、17.2%、19.0%、22.7%
排除异常值：10.6%（低）、22.7%（高）
中心聚类：16.4%、17.2%、19.0%
共识：16-19% 或 16-17%（保守）
```

### 预测共识

**方法：** 将共识 CAGR 应用于规模区间的中点

**示例：**
```
规模区间：150-220亿元 → 中点：185亿元
CAGR 共识：16-17%
按 16%：185 × (1.16)^6 = 451亿元
按 17%：185 × (1.17)^6 = 475亿元
共识预测：450-480亿元
```

## 舍入指南

这些是**典型惯例**——根据数值大小和模板样式进行调整：

| 数值类型 | 典型舍入 | 示例 |
|------------|------------------|---------|
| 大型市场规模（100亿+） | 最近 10亿 | 18.47 → 180亿 |
| 较小市场规模（<100亿） | 最近 5亿 或 1亿 | 2.3 → 25亿 |
| 规模区间 | 匹配来源精度 | 14.9-22.1 → 150-220亿 |
| CAGR | 整数% 或 0.5% | 16.4% → 16% 或 16.5% |
| 市场份额 | 最近 5% 或匹配来源 | 27.7% → 25% 或 30% |
| 收入（万元） | 1 位小数 | 18.47 → 1.85亿 |
| 倍数 | 1 位小数 | 9.688 → 9.7x |

**舍入原则：**
- 舍入不应实质性改变数值——对于较小的值，使用更精细的精度
- 一致性比精度更重要——类似数值使用相同舍入规则
- 创建区间时，低端向下舍入，高端向上舍入
- 对于汇总统计（均值、中位数），匹配输入数据的精度

## 验证清单

使用源数据中任何计算值之前：

### 公式验证
- [ ] 预测使用正确的 CAGR 公式：`PV × (1 + r)^n`
- [ ] 倍数计算为 EV ÷ 指标（非倒置）
- [ ] 增长率在分母中使用正确的基准年
- [ ] 百分比份额在适用情况下合计约 100%

### 输入验证
- [ ] 基准年数据与源文件匹配
- [ ] CAGR/增长率与所述来源方法论一致
- [ ] 时间段（n）计算正确
- [ ] 货币和单位一致（亿元 vs 万元）

### 输出验证
- [ ] 计算结果与来源所述数值匹配
- [ ] 如不匹配，调查方法论差异
- [ ] 舍入应用一致
- [ ] 结果合理（无数级错误）

### 共识验证
- [ ] 所有来源纳入区间计算
- [ ] 异常值排除方法论已记录
- [ ] 中点计算使用正确的平均方法
- [ ] 区间边界代表实际最小/最大值或已记录的子集

## 需调查的异常信号

**预测不匹配：**
- 计算预测与来源差异 >5%
- 可能原因：不同基准年、不同 CAGR 或舍入差异

**倍数不匹配：**
- 计算倍数与来源不同
- 可能原因：不同指标定义（LTM vs. NTM、收入 vs. 净收入）

**共识不匹配：**
- 你的共识与来源的共识不同
- 可能原因：来源排除了某些数据点、异常值处理方式不同

**如有疑问：** 在脚注中注明差异并展示你的计算方法论。
