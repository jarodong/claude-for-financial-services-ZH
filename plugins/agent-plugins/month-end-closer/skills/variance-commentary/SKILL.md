# 波动分析

给定当期实际数、上期实际数及同期预算，生成分析表。

## 阈值

当满足以下**任一**条件时，标记该行需分析：

- 绝对差异 ≥ 公司重要性阈值（使用提供的数值；默认为该行的 5% 或固定下限，取较大者）
- 该行在"必须分析"清单中（收入、人力成本、现金）

## 对每个标记行

| 列 | 内容 |
|---|---|
| **行项目** | 科目或项目名称 |
| **当期 / 上期 / 预算** | 三个数值 |
| **vs 上期差异** 和 **vs 预算差异** | 金额和百分比 |
| **驱动因素** | 一句话解释变动的底层业务原因——不是对数字的重述 |

驱动因素解释的是*为什么*，而非*是什么*："云计算支出增加 120 万元，因 5 月上线新增 GPU 预留实例"——而非"云计算支出增加 120 万元（18%）"。

## 驱动因素溯源

通过 internal-gl MCP 查看该行背后的业务活动（凭证来源分解、供应商构成、人员变动、量 × 单价）。如从数据中无法明确驱动因素，写"驱动因素不明确——标记交财务主管核查"，而非编造原因。

## 输出

分析表加一段简短叙述（3-5 句话），概括本期最大变动项目。
