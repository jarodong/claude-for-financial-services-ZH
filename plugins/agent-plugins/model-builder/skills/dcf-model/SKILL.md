## 关键约束——先读此处

这些约束适用于所有 DCF 模型构建。开始前审阅：

**环境：Office JS vs Python/openpyxl：**
- **如果在 Excel 中运行（Office 加载项 / Office JS 环境）：** 直接使用 Office JS——不要使用 Python/openpyxl。通过 `range.formulas = [["=D19*(1+$B$8)"]]` 写入公式。无需单独重算步骤；Excel 原生计算。使用 `range.format.*` 设置样式。同样的公式优先规则适用：对派生单元格设置 `.formulas`，绝不使用 `.values`。
- **如果生成独立 .xlsx 文件（无活动 Excel 会话）：** 使用 Python/openpyxl，然后在交付前运行 `recalc.py`。
- 此 Skill 其余部分使用 openpyxl 示例——在 Office JS 环境中转换为 API 调用，但所有原则（公式字符串、单元格注释、节检查点、敏感性表循环）完全相同。

**⚠️ Office JS 合并单元格陷阱：** 构建合并单元格的节标题时，不要先调用 `.merge()` 再在合并范围上设置 `.values`——Office JS 仍报告范围的原始尺寸，会抛出 `InvalidArgument: The number of rows or columns in the input array doesn't match the size or dimensions of the range`。改为先在左上角单元格写入值，然后合并和格式化整个范围：

```js
// 错误——抛出 InvalidArgument：
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.values = [["MARKET DATA & KEY INPUTS"]];  // 1×1 数组 vs 1×8 范围 → 失败

// 正确——先在单个单元格写值，再合并+格式化范围：
ws.getRange("A7").values = [["MARKET DATA & KEY INPUTS"]];
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.format.fill.color = "#1F4E79";
hdr.format.font.bold = true;
hdr.format.font.color = "#FFFFFF";
```

这适用于 DCF 中每个合并节标题（市场数据、情景块、现金流预测、终值、估值汇总、敏感性表）。

**公式优先于硬编码（不可妥协）：**
- 每个预测、利润率、折现因子、PV 和敏感性单元格必须是活动 Excel 公式——绝不是在 Python 中计算后写入的数字
- 使用 openpyxl 时：`ws["D20"] = "=D19*(1+$B$8)"` 正确；`ws["D20"] = calculated_revenue` 错误
- 唯一允许的硬编码数字是：(1) 原始历史输入，(2) 假设驱动因素（增长率、WACC 输入、终值 g），(3) 当前市场数据（股价、债务余额）
- 如果你发现自己在 Python 中计算某值并写入结果——停下。模型必须在用户更改假设时灵活响应。
- 生成独立 `.xlsx` 时，Python/openpyxl 只是写入器，不是估值引擎。DCF 估值、折现、终值、权益桥、每股价值和敏感性表输出必须落在 Excel 公式中；最终摘要引用这些公式输出，而不是 Python 预先算出的静态结果。

**文件路径和依赖纪律：**
- 生成脚本和 Excel 文件时使用绝对路径，避免因为前面切换到 Wind Skill 目录后，后续 `python3 Documents/xxx.py` 在错误目录执行。
- 不要无说明地安装依赖。若缺少 openpyxl、LibreOffice 或 `recalc.py`，先说明缺口，并给出继续生成、跳过重算或让用户安装依赖的选择。
- 最终交付时写清楚文件路径、数据来源、公式数量或重算结果；如果未完成公式重算，必须明确说明“尚未完成公式重算验证”。
- 如果对用户说“用 Python + openpyxl 生成 Excel”，必须同时说明“Python/openpyxl 只负责写入工作簿，DCF 计算在 Excel 公式中完成”。不能让用户误以为估值结果是 Python 算好后写入的静态表。

**逐步与用户验证（不要端到端构建）：**
- 数据检索后 → 向用户展示原始输入块（收入、利润率、股份数、净债务），在预测前确认
- 收入预测后 → 向用户展示预测顶线和增长率，在构建利润率前确认
- FCF 构建后 → 向用户展示完整 FCF 计划，在计算 WACC 前确认逻辑
- WACC 后 → 向用户展示计算和输入，在折现前确认
- 终值 + PV 后 → 向用户展示权益桥梁（EV → 权益价值 → 每股），在敏感性表前确认
- 每阶段尽早发现错误——在敏感性表建好后发现错误的利润率假设意味着重建所有下游

**敏感性分析表：**
- **使用奇数行和列**（标准：5×5，有时 7×7）——保证有真正的中心单元格
- **中心单元格 = 基准情景。** 构建轴值使中间行标题和列标题精确等于模型实际假设（如基准 WACC = 9.0%，中间行为 9.0%；终值 g = 3.0%，中间列为 3.0%）。中心单元格的输出必须等于模型实际隐含股价——这是表格构建正确的合理性检查。
- **高亮中心单元格**用中蓝填充（`#BDD7EE`）+ 粗体字，使基准情景立即可见。
- 填充所有单元格（通常 3 个表 × 25 个单元格 = 75）用完整 DCF 重算公式
- 使用 openpyxl 循环（或 Office JS 循环）程序化写入公式
- 无占位符文本，无线性近似，无需手动步骤
- 每个单元格必须为该假设组合重算完整 DCF

**单元格注释：**
- 在创建每个硬编码值时立即添加单元格注释
- 格式："来源：[系统/文档]，[日期]，[引用]，[URL 如适用]"
- 每个蓝色输入必须在进入下一节前有注释
- 不要推迟到末尾或写"TODO: add source"

**模型布局规划：**
- 在写任何公式前定义所有节行位置
- 先写所有标题和标签
- 再写所有节分隔符和空行
- 然后使用锁定的行位置写公式
- 创建后立即测试公式

**公式重算：**
- 交付前运行 `python recalc.py model.xlsx 30`
- 修复所有错误直到状态为"success"
- 要求零公式错误（#REF!、#DIV/0!、#VALUE! 等）

**情景块：**
- 为悲观/基准/乐观情景创建单独块
- 在每个块中水平显示预测年份的假设
- 使用 IF 公式：`=IF($B$6=1,[悲观单元格],IF($B$6=2,[基准单元格],[乐观单元格]))`
- 验证公式引用正确的情景块单元格

## DCF 流程工作流

### 第 1 步：数据检索与验证

从 MCP 服务器、Wind MCP Skill、用户提供的数据和必要的公开资料获取数据。

**数据来源优先级：**
1. **MCP 服务器**（如已配置）——来自 Wind / iFinD 等提供商的结构化财务数据。
2. **Wind MCP Skill CLI**（如存在）——检查 `~/.agents/skills/wind-mcp-skill`，并在该目录下调用 `node scripts/cli.mjs call ...`。对 A 股公司，这是优先级很高的数据通道。
3. **用户提供的数据**——来自其研究的历史财务数据。
4. **公开网页 / Web Search**——仅在 MCP 和 Wind MCP Skill 均不可用，或用户明确同意时作为后备。

**A 股 / 中国公司强制要求：**
- 开始 DCF 前，先尝试通过 Wind MCP 或 Wind MCP Skill 获取历史财务、行情、估值倍数和风险指标。
- 可使用 Wind MCP Skill CLI 示例：
  - `stock_data get_stock_fundamentals`：营业收入、净利润、EBITDA、毛利率、净利率、ROE、资本支出、折旧摊销。
  - `stock_data get_stock_price_indicators`：股价、市值、估值倍数。
  - `stock_data get_risk_metrics`：Beta、波动率。
- Wind CLI 调用应分批进行，先取目标公司核心数据，再取可比公司和估值倍数；不要为了“并行探索”同时发起大量命令。
- 如果 Wind MCP 不可用，先向用户说明并询问是否安装/启用 Wind MCP，或是否允许临时使用公开网页数据。
- 不要在未说明数据源不可用的情况下直接 Web Search。

**验证清单：**
- 验证净债务 vs 净现金（对估值关键）
- 确认稀释后流通股数（检查近期回购/发行）
- 验证历史利润率与商业模式一致
- 交叉检查收入增长率与行业基准
- 验证税率合理（通常 21-28%）

### 第 2 步：历史分析（3-5 年）

分析并记录：
- **收入增长趋势**：计算 CAGR，识别驱动因素
- **利润率变化**：跟踪毛利率、EBIT 利润率、FCF 利润率
- **资本密集度**：D&A 和资本支出占收入百分比
- **营运资本效率**：NWC 变化占收入增长百分比
- **回报指标**：ROIC、ROE 趋势

创建汇总表显示：
```
历史指标（LTM）：
收入：X 百万元
收入增长：X% CAGR
毛利率：X%
EBIT 利润率：X%
D&A 占收入：X%
资本支出占收入：X%
FCF 利润率：X%
```

### 第 3 步：构建收入预测

**方法论：**
1. 从最新实际收入开始（LTM 或最近财年）
2. 对每个预测年份应用增长率
3. 同时显示人民币金额和计算的增长率%

**增长率框架：**
- 第 1-2 年：较高增长反映近期可见性
- 第 3-4 年：逐步向行业平均收敛
- 第 5 年+：趋近终值增长率

**公式结构：**
- 收入(第 N 年) = 收入(第 N-1 年) × (1 + 增长率)
- 增长%(第 N 年) = 收入(第 N 年) / 收入(第 N-1 年) - 1

**三情景方法：**
```
悲观情景：保守增长（如 8-12%）
基准情景：最可能情景（如 12-16%）
乐观情景：乐观增长（如 16-20%）
```

### 第 4 步：运营费用建模

**固定/可变成本分析：**

运营费用应建模现实的运营杠杆：
- **销售与营销**：通常为收入的 15-40%，取决于商业模式
- **研发**：科技公司通常 10-30%
- **一般及行政**：通常为收入的 8-15%，随规模显示杠杆效应

**关键原则：**
- 所有百分比基于收入，非毛利
- 建模运营杠杆：百分比应随收入规模下降
- S&M、R&D、G&A 保持单独行项目
- 计算 EBIT = 毛利 - 总运营费用

**利润率扩张框架：**
```
当前状态 → 目标状态（第 5 年）
毛利率：X% → Y%（基于规模、效率论证）
EBIT 利润率：X% → Y%（收入增长 + 运营杠杆的结果）
```

### 第 5 步：自由现金流计算

**按正确顺序构建 FCF：**

```
EBIT
(-) 税务（EBIT × 税率）
= NOPAT（税后净营业利润）
(+) D&A（非现金费用，收入百分比）
(-) 资本支出（收入百分比，通常 4-8%）
(-) Δ NWC（营运资本变化）
= 无杠杆自由现金流
```

**营运资本建模：**
- 按收入变化百分比计算（增量收入）
- 典型范围：收入变化的 -2% 到 +2%
- 负数 = 现金来源（营运资本释放）
- 正数 = 现金使用（营运资本积累）

**维护性 vs 增长性资本支出：**
- 维护性资本支出：维持当前运营（约 2-3% 收入）
- 增长性资本支出：支持扩张（额外 2-5% 收入）
- 总资本支出应与公司增长战略一致

### 第 6 步：资本成本（WACC）研究

**CAPM 股权成本方法论：**

```
股权成本 = 无风险利率 + Beta × 股权风险溢价

其中：
- 无风险利率 = 当前 10 年期国债收益率
- Beta = 5 年月度股票 Beta vs 市场指数
- 股权风险溢价 = 5.0-6.0%（市场标准）
```

**债务成本计算：**

```
税后债务成本 = 税前债务成本 × (1 - 税率)

从以下确定税前债务成本：
- 信用评级（如有）
- 公司债券当前收益率
- 利息费用 / 财务报表中的总债务
```

**资本结构权重：**

```
权益市值 = 当前股价 × 流通股数
净债务 = 总债务 - 现金及等价物
企业价值 = 市值 + 净债务

权益权重 = 市值 / 企业价值
债务权重 = 净债务 / 企业价值

WACC = (股权成本 × 权益权重) + (税后债务成本 × 债务权重)
```

**特殊情况：**
- **净现金头寸**：如果现金 > 债务，净债务为负
  - 债务权重可能为负
  - WACC 计算相应调整
- **无债务**：WACC = 股权成本

**典型 WACC 范围：**
- 大盘稳定公司：7-9%
- 成长型公司：9-12%
- 高增长/高风险：12-15%

### 第 7 步：折现率应用（5-10 年预测）

**年中惯例：**
- 假设现金流在年中发生
- 折现期间：0.5、1.5、2.5、3.5、4.5 等
- 折现因子 = 1 / (1 + WACC)^期间

**现值计算：**
```
对每个预测年份：
FCF 现值 = 无杠杆 FCF × 折现因子

示例（第 1 年）：
FCF = 1,000万元
WACC = 10%
期间 = 0.5
折现因子 = 1 / (1.10)^0.5 = 0.9535
PV = 1,000万元 × 0.9535 = 954万元
```

**预测期选择：**
- **5 年**：多数分析的标准
- **7-10 年**：有更长跑道的高增长公司
- **3 年**：成熟稳定企业

### 第 8 步：终值计算

**永续增长法（首选）：**

```
终值 FCF = 最后一年 FCF × (1 + 终值增长率)
终值 = 终值 FCF / (WACC - 终值增长率)

关键约束：终值增长率 < WACC（否则价值无穷大）
```

**终值增长率选择：**
- 保守：2.0-2.5%（GDP 增长率）
- 中等：2.5-3.5%
- 激进：3.5-5.0%（仅对市场领导者）

**不要超过**：无风险利率或长期 GDP 增长

**退出倍数法（替代）：**
```
终值 = 最后一年 EBITDA × 退出倍数

其中退出倍数来自：
- 行业可比交易倍数
- 可比交易倍数
- 典型范围：8-15x EBITDA
```

**终值现值：**
```
终值现值 = 终值 / (1 + WACC)^最终期间

其中最终期间考虑时间：
5 年模型年中惯例：期间 = 4.5
```

**终值合理性检查：**
- 应占企业价值的 50-70%
- 如果 >75%，模型可能过度依赖终值假设
- 如果 <40%，检查终值假设是否过于保守

### 第 9 步：企业价值到权益价值桥梁

**估值汇总结构：**

```
(+) 预测 FCF 现值合计 = X 百万元
(+) 终值现值 = Y 百万元
= 企业价值 = Z 百万元

(-) 净债务 [或 + 净现金如为负] = A 百万元
= 权益价值 = B 百万元

÷ 稀释后流通股 = C 百万股
= 隐含每股价格 = XX.XX元

当前股价 = YY.YY元
隐含回报 = (隐含价格 / 当前价格) - 1 = XX%
```

**关键调整：**
- **净债务 = 总债务 - 现金及等价物**
  - 如果为正：从 EV 中减出（减少权益价值）
  - 如果为负（净现金）：加到 EV（增加权益价值）
- **使用稀释后股份数**：包括期权、RSU、可转换证券
- **其他调整**（如适用）：
  - 少数股东权益
  - 养老金负债
  - 经营租赁义务

**估值输出格式：**
```csv
估值组成部分,金额（百万元）
预测 FCF 现值,X.X
终值现值,Y.Y
企业价值,Z.Z
(-) 净债务,A.A
权益价值,B.B
,,
流通股数（百万）,C.C
隐含每股价格,XX.XX元
当前股价,YY.YY元
隐含上行/(下行),+XX%
```

### 第 10 步：敏感性分析

在 DCF 工作表底部构建**三个敏感性表**，显示估值如何随不同假设变化：

1. **WACC vs 终值增长率**——显示企业价值对折现率和永续增长的敏感性
2. **收入增长 vs EBIT 利润率**——显示顶线增长和运营杠杆的影响
3. **Beta vs 无风险利率**——显示对股权成本组件的敏感性

**实现方式**：这些是简单的二维网格（不是 Excel 的"数据表"功能），每个单元格包含公式。每个单元格必须包含该特定假设组合的完整 DCF 重算。参见关键约束节了解使用 openpyxl 程序化填充所有 75 个单元格的详细要求。

### 情景块选择模式——遵循此方法

**假设按情景组织在单独块中：**

**关键结构——每节标题三行：**

```csv
悲观情景假设（节标题，跨列合并）
假设,FY1,FY2,FY3,FY4,FY5
收入增长（%）,12%,10%,9%,8%,7%
EBIT 利润率（%）,45%,44%,43%,42%,41%

基准情景假设（节标题，跨列合并）
假设,FY1,FY2,FY3,FY4,FY5
收入增长（%）,16%,14%,12%,10%,9%
EBIT 利润率（%）,48%,49%,50%,51%,52%

乐观情景假设（节标题，跨列合并）
假设,FY1,FY2,FY3,FY4,FY5
收入增长（%）,20%,18%,15%,13%,11%
EBIT 利润率（%）,50%,51%,52%,53%,54%
```

**每个情景块必须有列标题行**显示预测年份（FY2025E、FY2026E 等），在节标题正下方。没有这个，用户无法判断哪个假设值对应哪一年。

**如何引用假设——创建汇总列：**
1. 情景选择器单元格（如 B6）包含 1=悲观、2=基准、或 3=乐观
2. 创建汇总列，使用 INDEX 或 OFFSET 公式从正确情景块提取
3. 预测公式引用汇总列（干净的单元格引用）
4. 每个情景块包含完整的 DCF 假设集，横跨预测年份

**推荐的汇总列模式（使用 INDEX）：**
`=INDEX(B10:D10, 1, $B$6)`

**不要这样做——分散的 IF 语句：**
`=IF($B$6=1,[悲观块单元格],IF($B$6=2,[基准块单元格],[乐观块单元格]))`

汇总列方法集中逻辑，使模型更易审计。

### 正确的收入预测模式

**使用 INDEX 公式创建汇总列，然后在预测中引用：**

**第 1 步——FY1 增长的汇总列：**
`=INDEX([悲观 FY1 增长]:[乐观 FY1 增长], 1, $B$6)`

**第 2 步——收入预测引用汇总列：**
`收入第 1 年: =D29*(1+$E$10)`

其中：
- D29 = 上年收入
- $E$10 = FY1 增长的汇总列单元格（包含 INDEX 公式）
- $B$6 = 情景选择器（1=悲观，2=基准，3=乐观）

**此方法比在每个预测公式中嵌套 IF 语句更干净**，且更容易审计正在使用的情景假设。

### 正确的 FCF 公式模式

**使用 INDEX 公式的汇总列，然后在 FCF 计算中引用：**

**汇总列方法：**
```csv
项目,公式,引用
D&A,=E29*$E$21,$E$21 = D&A% 的汇总列
资本支出,=E29*$E$22,$E$22 = 资本支出% 的汇总列
Δ NWC,=(E29-D29)*$E$23,$E$23 = NWC% 的汇总列
无杠杆 FCF,=E57+E58-E60-E62,E57=NOPAT E58=D&A E60=资本支出 E62=Δ NWC
```

**每个汇总列单元格包含 INDEX 公式**，基于情景选择器从适当情景块提取。这保持预测公式干净可审计。

写公式前，确认情景块行位置并设置汇总列。

### 正确的敏感性表实现

**重要**：这些不是 Excel 的"数据表"功能。这些是简单网格，你用 openpyxl 写常规公式。是的，这意味着总共约 75 个公式（3 个表 × 25 个单元格），但这很直接且必需。

**用公式程序化填充：**

每个敏感性表必须完全填充公式，为每个假设组合重算隐含股价。**不要使用 Excel 的数据表功能**（需要手动干预，无法通过 openpyxl 自动化）。

**实现方法——具体示例：**

**表结构——5×5 网格（奇数维度，基准情景居中）：**

如果模型基准 WACC = 9.0%，基准终值增长 = 3.0%，围绕这些值对称构建轴：

```csv
WACC vs 终值增长率,  2.0%,  2.5%,  3.0%,  3.5%,  4.0%
              8.0%,   [公式],[公式],[公式],[公式],[公式]
              8.5%,   [公式],[公式],[公式],[公式],[公式]
              9.0%,   [公式],[公式],[★  ],[公式],[公式]  ← 中间行 = 基准 WACC
              9.5%,   [公式],[公式],[公式],[公式],[公式]
             10.0%,   [公式],[公式],[公式],[公式],[公式]
                                  ↑
                         中间列 = 基准终值 g
```

**★ = 中心单元格。** 其公式输出必须等于模型实际隐含股价（来自估值汇总）。对此单元格应用中蓝填充（`#BDD7EE`）和粗体字，使基准情景视觉锚定。

**轴值规则：** `axis_values = [base - 2*step, base - step, base, base + step, base + 2*step]`——围绕基准对称，奇数保证有中心。

**公式模式——单元格 B88（WACC=8.0%，终值增长率=2.0%）：**

B88 中的公式应使用以下值重算隐含价格：
- 行标题的 WACC：`$A88`（8.0%）
- 列标题的终值增长率：`B$87`（2.0%）

**推荐方法：** 引用主 DCF 计算但替换这些值。

**示例公式结构：**
`=([使用 $A88 作为折现率的 PV FCF 合计] + [使用 B$87 作为增长率和 $A88 作为 WACC 的终值] - [净债务]) / [股份数]`

**关键——为 5×5 网格的每个单元格写公式（每表 25 个单元格，共 75 个）。** 使用 openpyxl 在循环中程序化写入这些公式。不要跳过此步骤或留占位符文本。

**Python 实现模式：**
```python
# 填充敏感性表的伪代码
for row_idx, wacc_value in enumerate(wacc_range):
    for col_idx, term_growth_value in enumerate(term_growth_range):
        # 构建使用 wacc_value 和 term_growth_value 的公式
        formula = f"=<DCF 重算使用 {wacc_value} 和 {term_growth_value}>"
        ws.cell(row=start_row+row_idx, column=start_col+col_idx).value = formula
```

**敏感性表在用户打开文件时必须立即工作，无需手动步骤。**

<common_mistakes>

本节包含构建 DCF 模型时所有应避免的错误模式。

### 错误：简化敏感性表近似或占位符文本

**不要使用线性近似：**

```
// 错误 - 线性近似
B97: =B88*(1+(0.096-0.116))    // 假设线性关系

// 错误 - 除法捷径
B105: =B88/(1+(E48-0.07))      // 不重算完整 DCF
```

**不要留占位符文本：**
```
// 错误 - 占位符说明
"说明：使用 Excel 数据表功能（数据 → 假设分析 → 数据表）填充敏感性表。"

// 错误 - 空单元格
[留空因为"这很复杂"]
```

**不要混淆术语：**
- ❌ "敏感性表需要 Excel 的数据表功能"（不——那是我们无法使用的特定 Excel 工具）
- ✅ "敏感性表是每个单元格有公式的简单网格"（是——这是我们要构建的）

**这些捷径为何错误：**
- 线性近似公式实际上不重算 DCF——只是应用简单数学调整
- 关系不是线性的，结果会不准确
- 占位符文本需要用户手动干预
- 模型交付时不能立即使用
- 不专业或不可交付给客户
- 空单元格 = 不完整交付物

**常见合理化借口——拒绝：**
"写 75+ 个公式感觉很复杂，所以我留个说明让用户手动完成。"

**现实：** 使用 Python 的 openpyxl 写循环时，75 个公式很直接。每个公式遵循相同模式——只需替换行列值。这是交付物的必需部分。

**改为：** 用为该特定假设组合重算完整 DCF 的公式填充每个敏感性单元格

### 错误：缺少单元格注释

**不要这样做：**
- 创建所有硬编码输入不加注释
- 想着"稍后添加"
- 写"TODO: add source"
- 留蓝色输入无文档

**为何错误：**
- 无法验证数据来源
- 不满足 xlsx Skill 要求
- 不具备审计条件
- 后期修复浪费时间

**改为：** 在创建每个硬编码值时立即添加单元格注释

### 错误：公式行引用偏移

**症状：**
FCF 节引用了错误的假设行：
`D&A:  =E29*$E$34    // 应该是 $E$21，但引用了错误行`
`资本支出: =E29*$E$41   // 应该是 $E$22，但行偏移了`

**为何发生：**
1. 先写了公式
2. 然后插入了标题
3. 所有行引用偏移
4. 现在公式指向错误单元格 → #REF! 错误

**改为：** 先锁定行布局，再写公式

### 错误：跨情景的单行假设

**不要这样组织假设：**
```csv
假设,悲观,基准,乐观
收入增长 FY1,10%,13%,16%
收入增长 FY2,9%,12%,15%
```
这种垂直布局难以看到每个情景中跨年份的变化。

**为何错误：**
- 难以看到每个情景中跨年份假设的演变
- 难以比较跨完整预测期的情景假设
- 审查情景逻辑不够直观

**改为：**
- 为每个情景（悲观、基准、乐观）创建单独块
- 在每个块中，水平显示跨预测年份的假设
- 这使每个情景的假设作为连贯集合更易审查

### 错误：无边框

**不要交付无边框模型：**
- 无节界定
- 所有单元格混在一起
- 难读且不专业

**为何错误：**
- 不可交付给客户
- 难以导航
- 看起来业余

**改为：** 在所有主要节周围添加边框

### 错误：字体颜色错误或无字体颜色区分

**不要这样做：**
- 所有文本为黑色
- 仅使用填充颜色（无字体颜色变化）
- 搞混哪些单元格应为蓝色 vs 黑色

**为何错误：**
- 无法区分输入和公式
- 审计变得不可能
- 违反 xlsx Skill 要求

**改为：** 所有硬编码输入蓝色文本，所有公式黑色文本，工作表链接绿色文本

### 错误：运营费用基于毛利

**不要这样做：**
`S&M: =E33*0.15    // E33 = 毛利（错误）`

**为何错误：**
- 运营费用随收入规模，非毛利
- 产生不现实的利润率变化
- 不是企业实际运营方式

**改为：**
`S&M: =E29*0.15    // E29 = 收入（正确）`

### 五大错误总结

1. **公式行引用偏移** → 在写公式前定义所有行位置
2. **缺少单元格注释** → 在创建单元格时添加注释，非末尾
3. **简化敏感性表** → 用完整 DCF 重算公式填充所有单元格，非近似
4. **情景块引用错误** → 确保 IF 公式从正确的悲观/基准/乐观块提取
5. **无边框** → 添加专业节边框以达到可交付给客户的外观

此外，注意这些错误：

### WACC 计算错误
- 资本结构中混淆账面价值和市场价值
- 错误使用股权 Beta 而非资产/无杠杆 Beta
- 对债务成本应用错误税率
- 无风险利率不正确（必须使用当前 10 年期国债）
- 未调整净债务 vs 净现金头寸

### 增长假设缺陷
- 终值增长率 > WACC（产生无穷大价值）
- 预测增长率与历史表现不一致
- 忽略行业增长约束
- 收入增长与单位经济模型不一致
- 无运营论证的利润率扩张

### 终值错误
- 使用错误的增长方法（永续 vs 退出倍数）
- 终值 > 企业价值的 80%（暗示过度依赖）
- 终值利润率与稳态假设不一致
- 终值的折现期间错误

### 现金流预测错误
- 运营费用基于毛利而非收入
- D&A/资本支出百分比与商业模式不匹配
- 营运资本变化计算不正确
- 年份间税率不一致
- NOPAT 计算错误

**这些错误最常见。开始任何 DCF 构建前重读此节。**

## Excel 模型结构

### 工作表架构

创建**两个工作表**：

1. **DCF**——主估值模型，敏感性分析在底部
2. **WACC**——资本成本计算

**关键**：敏感性表放在 DCF 工作表底部（非单独工作表）。这保持所有估值输出在一起。

### 公式重算（强制）

创建或修改 Excel 模型后，使用 xlsx Skill 的 recalc.py 脚本**重算所有公式**：

```bash
python recalc.py [Excel文件路径] [超时秒数]
```

示例：
```bash
python recalc.py AAPL_DCF_Model_2025-10-12.xlsx 30
```

脚本将：
- 使用 LibreOffice 重算所有工作表中的所有公式
- 扫描所有单元格的 Excel 错误（#REF!、#DIV/0!、#VALUE!、#NAME?、#NULL!、#NUM!、#N/A）
- 返回详细 JSON 含错误位置和计数

**预期输出格式：**
```json
{
  "status": "success",           // 或 "errors_found"
  "total_errors": 0,              // 总错误数
  "total_formulas": 42,           // 文件中公式数
  "error_summary": {}             // 仅在发现错误时存在
}
```

**如果发现错误**，输出将包含详情：
```json
{
  "status": "errors_found",
  "total_errors": 2,
  "total_formulas": 42,
  "error_summary": {
    "#REF!": {
      "count": 2,
      "locations": ["DCF!B25", "DCF!C25"]
    }
  }
}
```

**修复所有错误**并重新运行 recalc.py 直到状态为"success"再交付模型。

### 格式标准

**重要**：遵循 xlsx Skill 的公式构建规则和数字格式约定。DCF Skill 添加特定的视觉呈现标准。

**颜色方案——两层**：

**第 1 层：字体颜色（xlsx Skill 强制）**
- **蓝色文本（RGB: 0,0,255）**：所有硬编码输入（股价、股份数、历史数据、假设）
- **黑色文本（RGB: 0,0,0）**：所有公式和计算
- **绿色文本（RGB: 0,128,0）**：链接到其他工作表（WACC 工作表引用）

**第 2 层：填充颜色——专业蓝灰色调（默认，除非用户另行指定）**
- **保持极简**——仅使用蓝色和灰色作为填充。不要引入绿色、黄色、橙色或多种强调色。颜色过多的模型显得业余。
- **默认填充色板：**
  - **节标题**：深蓝（RGB: 31,78,121 / `#1F4E79`）背景配白色粗体文本
  - **子标题/列标题**：浅蓝（RGB: 217,225,242 / `#D9E1F2`）背景配黑色粗体文本
  - **输入单元格**：浅灰（RGB: 242,242,242 / `#F2F2F2`）背景配蓝色字体——或白色背景配蓝色字体以实现最大极简
  - **计算单元格**：白色背景配黑色字体
  - **输出/汇总行**（每股价值、EV 等）：中蓝（RGB: 189,215,238 / `#BDD7EE`）背景配黑色粗体字
- **就这样——3 种蓝色 + 1 种灰色 + 白色。** 忍住添加更多的冲动。
- 用户提供的模板或明确颜色偏好始终覆盖这些默认值。

**两层如何配合：**
- 输入单元格：蓝色字体 + 浅灰填充 = "硬编码输入"
- 公式单元格：黑色字体 + 白色背景 = "计算值"
- 工作表链接：绿色字体 + 白色背景 = "来自另一工作表的引用"
- 关键输出：黑色粗体字 + 中蓝填充 = "这是答案"

**字体颜色告诉你它*是*什么（输入/公式/链接）。填充颜色告诉你*在*哪里（标题/数据/输出）。**

### 边框标准（专业外观必需）

**粗边框**（1.5pt）围绕主要节：
- 关键输入节
- 预测假设节
- 5 年现金流预测节
- 终值节
- 估值汇总节
- 每个敏感性分析表

**中等边框**（1pt）在子节之间：
- 公司详情 vs 历史表现
- 增长假设 vs EBIT 利润率 vs FCF 参数

**细边框**（0.5pt）围绕数据表：
- 情景假设表（悲观 | 基准 | 乐观 | 选定）
- 历史 vs 预测财务矩阵

**无边框：** 表内个别单元格（保持干净、可扫描）

**边框是强制的**——没有专业边框的模型不可交付给客户。

**数字格式**（遵循 xlsx Skill 标准）：
- **年份**：格式化为文本字符串（如 "2024" 而非 "2,024"）
- **百分比**：`0.0%`（一位小数）
- **货币**：百万用 `¥#,##0`；每股用 `¥#,##0.00`——始终在标题中指定单位（"收入（百万元）"）
- **零值**：使用数字格式使所有零显示为"-"（如 `¥#,##0;(¥#,##0);-`）
- **大数字**：`#,##0` 带千位分隔符
- **负数**：`(#,##0)` 用括号（非减号）

**单元格注释（所有硬编码输入强制）**：

按 xlsx Skill，所有硬编码值必须有单元格注释记录来源。格式："来源：[系统/文档]，[日期]，[引用]，[URL 如适用]"

**关键**：在创建单元格时立即添加注释。不要推迟到末尾。

**关键 WACC 公式：**
```
市值 = 价格 × 股份数
净债务 = 总债务 - 现金
企业价值 = 市值 + 净债务
权益权重 = 市值 / EV
债务权重 = 净债务 / EV
WACC = (股权成本 × 权益权重) + (税后债务成本 × 债务权重)
```

### 敏感性分析（DCF 工作表底部）

**术语提醒**："敏感性表" = 带行列标题和每个数据单元格公式的简单二维网格。不是 Excel 的"数据表"功能（数据 → 假设分析 → 数据表）。你将使用 openpyxl 向每个单元格写常规 Excel 公式。

**位置**：DCF 工作表第 87 行起（非单独工作表）

**三个敏感性表，垂直堆叠：**

1. **WACC vs 终值增长率**（第 87-100 行）——5×5 网格 = 25 个公式单元格
2. **收入增长 vs EBIT 利润率**（第 102-115 行）——5×5 网格 = 25 个公式单元格
3. **Beta vs 无风险利率**（第 117-130 行）——5×5 网格 = 25 个公式单元格

**需写的公式总数：75**（此为必需，非可选）

**关键**：所有敏感性表单元格必须使用 openpyxl 程序化填充公式。不要使用线性近似捷径。不要留占位符文本或手动步骤说明。不要以"复杂"为由合理化留空单元格——用 Python 循环生成公式。

**表设置：**
1. 创建带行列标题的表结构（要测试的假设值）
2. 为每个数据单元格填充公式：
   - 使用行标题值（如 WACC = 9.0%）
   - 使用列标题值（如终值增长率 = 3.0%）
   - 用这些特定假设重算完整 DCF
   - 返回该情景的隐含股价
3. 交付时所有单元格必须包含有效公式
4. 用条件格式化单元格：高值绿色渐变，低值红色渐变
5. 粗体基准情景单元格
6. 表间留 1-2 空行

**无需手动干预**——敏感性表在用户打开文件时必须完全可用。

## 情景选择器实现

**三情景框架：**

### 悲观情景
- 保守收入增长（历史范围低端）
- 利润率压缩或无扩张
- 更高 WACC（风险溢价增加）
- 更低终值增长率
- 更高资本支出假设

### 基准情景
- 一致预期或管理层指引收入增长
- 基于运营杠杆的适度利润率扩张
- 当前市场隐含 WACC
- GDP 对齐的终值增长（2.5-3.0%）
- 标准资本支出假设

### 乐观情景
- 乐观收入增长（预测高端）
- 显著利润率扩张
- 更低 WACC（降低风险溢价）
- 更高终值增长（3.5-5.0%）
- 降低资本支出强度

**公式实现：**

**不要使用分散嵌套 IF 公式。** 改为创建汇总列，使用 INDEX 或 OFFSET 公式从适当情景块提取。

**推荐模式（使用 INDEX）：**
`=INDEX(B10:D10, 1, $B$6)` 其中 `B10:D10` = 悲观/基准/乐观值，`1` = 行偏移，`$B$6` = 情景选择器单元格（1、2 或 3）

**然后在所有预测中引用汇总列：**
`收入第 1 年: =D29*(1+$E$10)` 其中 $E$10 是 FY1 增长的汇总列值。

此方法集中情景逻辑，使模型更易审计和维护。

## 常见变体

### 高增长科技公司
- 更长预测期（7-10 年）
- 更高初始增长率（20-30%）
- 随时间显著利润率扩张
- 更高 WACC（12-15%）
- 建模单位经济（用户、ARPU 等）

### 成熟/稳定公司
- 更短预测期（3-5 年）
- 温和增长率（GDP +1-3%）
- 稳定利润率
- 更低 WACC（7-9%）
- 关注现金生成和资本配置

### 周期性公司
- 跨经济周期建模
- 周期中段正常化利润率
- 考虑谷底和峰值情景
- 调整 Beta 反映周期性

### 多业务公司
- 为每个业务单元单独 DCF
- 不同业务不同增长率和利润率
- 分部加总估值
- 考虑协同效应
