# 审计电子表格

审计公式和数据的准确性与错误。范围决定深度——从选区的快速公式检查到完整的财务模型完整性审计。

## 第一步：确定范围

如果用户已给出范围，直接使用。否则**询问用户**：

> 你想让我审计什么范围？
> - **选区** —— 仅当前选中的区域
> - **工作表** —— 仅当前活动工作表
> - **模型** —— 整个工作簿，含财务模型完整性检查（BS 勾稽、现金勾稽、结转、逻辑合理性）

**模型**范围是最深层的——用于 DCF、LBO、三表、并购、Comps 或任何集成财务模型，在发送给客户或投决会之前使用。

---

## 第二步：公式级检查（所有范围）

无论范围如何，都运行以下检查：

| 检查项 | 关注内容 |
|--------|----------|
| 公式错误 | `#REF!`、`#VALUE!`、`#N/A`、`#DIV/0!`、`#NAME?` |
| 公式中的硬编码 | `=A1*1.05` —— `1.05` 应该是单元格引用 |
| 不一致的公式 | 行/列中与相邻公式模式不同的公式 |
| 范围偏移 | `SUM`/`AVERAGE` 遗漏了第一行或最后一行 |
| 覆盖粘贴的公式 | 看起来像公式但实际是硬编码值的单元格 |
| 循环引用 | 有意或无意的 |
| 断裂的跨表链接 | 引用了已移动或删除的单元格 |
| 单位/规模不匹配 | 千与百万混用，% 存储为整数 |
| 隐藏行/工作表 | 可能包含覆盖或过时计算 |

## 第三步：模型完整性检查（仅模型范围）

如果范围是**模型**，识别模型类型（DCF / LBO / 三表 / 并购 / Comps / 自定义），并运行以下适当的完整性检查。

### 3a. 结构审查

| 检查项 | 关注内容 |
|--------|----------|
| 输入/公式分离 | 输入是否与计算明确分离？ |
| 颜色规范 | 蓝色=输入、黑色=公式、绿色=链接——或模型使用的规范，是否一致应用？ |
| 工作表流程 | 逻辑顺序（假设 → 利润表 → 资产负债表 → 现金流量表 → 估值）？ |
| 日期标题 | 所有工作表一致？ |
| 单位 | 一致（千 vs 百万 vs 实际数）？ |

### 3b. 资产负债表

| 检查项 | 测试 |
|--------|------|
| BS 勾稽 | 总资产 = 总负债 + 权益（每期） |
| RE 结转 | 期初 RE + 净利润 − 股利 = 期末 RE |
| 商誉/无形资产 | 流转来自收购假设（如为并购） |

如果 BS 不勾稽，**量化每期差异并追溯断裂点**——在修复之前，其他一切都不可信。

### 3c. 现金流量表

| 检查项 | 测试 |
|--------|------|
| 现金勾稽 | CF 期末现金 = BS 现金（每期） |
| CF 加总 | CFO + CFI + CFF = Δ 现金 |
| D&A 一致 | CF 上的 D&A = 利润表上的 D&A |
| CapEx 一致 | CF 上的 CapEx 与 BS 上的 PP&E 结转匹配 |
| 营运资本变动 | 符号与 BS 变动一致（ΔAR、ΔAP、ΔInventory） |

### 3d. 利润表

| 检查项 | 测试 |
|--------|------|
| 收入构建 | 与分部/产品明细勾稽 |
| 所得税 | 所得税费用 = 税前利润 × 税率（允许递延所得税调整） |
| 股份数 | 与稀释计划一致（期权、可转债、回购） |

### 3e. 循环引用

- 利息 → 债务余额 → 现金 → 利息是 LBO/三表模型中常见的有意循环
- 如为有意：验证迭代开关存在且有效
- 如为无意：追溯循环并标记如何打断

### 3f. 逻辑与合理性

| 检查项 | 标记条件 |
|--------|----------|
| 增长率 | >100% 收入增长无解释 |
| 利润率 | 超出行业正常范围 |
| 终值主导 | TV > DCF EV 的约 75%（黄旗） |
| 曲棍球棒 | 预测在远期不切实际地陡升 |
| 复利效应 | EBITDA 到第 10 年复利至荒谬金额 |
| 边界情况 | 模型在 0% 或负增长、负 EBITDA、杠杆为负时崩溃 |

### 3g. 模型类型特定 Bug

**DCF：**
- 折现率应用于错误期间（年中 vs 年末）
- 终值未折现回来
- WACC 使用账面价值而非市场价值
- FCF 包含利息费用（应为无杠杆）
- 税盾重复计算

**LBO：**
- 债务偿还与现金回收机制不匹配
- PIK 利息未累积到本金
- 管理层跟投未反映在回报中
- 退出倍数应用于错误 EBITDA（LTM vs NTM）
- 费用未从 Day 1 权益中扣除

**并购：**
- 增厚/摊薄使用错误股份数（交易前 vs 交易后）
- 协同效应未分阶段注入
- 购买价格分配不平衡
- 未包含现金的放弃利息
- 交易费用未纳入来源与用途

**三表：**
- 营运资本变动符号错误
- 折旧与 PP&E 计划不匹配
- 债务到期计划与本金支付不匹配
- 股利超过净利润无解释

## 第四步：报告

输出发现表格：

| # | 工作表 | 单元格/区域 | 严重程度 | 类别 | 问题 | 建议修复 |
|---|--------|-------------|----------|------|------|----------|

**严重程度：**
- **Critical** —— 输出错误（BS 不勾稽、公式断裂、现金不勾稽）
- **Warning** —— 有风险（硬编码、不一致公式、边界情况失败）
- **Info** —— 风格/最佳实践（颜色编码、布局、命名）

对于**模型**范围，前置摘要行：

> 模型类型：[DCF/LBO/三表/...] —— 总体：[Clean / Minor Issues / Major Issues] —— [N] 个 critical，[N] 个 warning，[N] 个 info

**不要在未询问的情况下修改任何内容** —— 先报告，按要求修复。

---

## 注意事项

- **BS 勾稽优先** —— 如果不勾稽，下游一切都可疑
- **硬编码覆盖是静默 Bug 的头号来源** —— 积极搜索
- **符号规范错误**（现金流出用正数还是负数）极其常见
- 如果模型使用 VBA 宏，标注任何无法从公式审计的宏驱动计算
