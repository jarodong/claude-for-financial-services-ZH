# 晨会纪要

## 工作流程

### 第一步：隔夜动态

扫描覆盖范围内的相关事件：

**财报与指引**
- 是否有覆盖公司在隔夜或盘前发布财报？
- 财报超预期/不及预期（收入、EPS、关键指标）
- 指引变化（上调、下调、维持）

**新闻与事件**
- 并购公告或传闻
- 管理层变动
- 产品发布或监管决策
- 竞争对手分析师升级/降级
- 影响行业的宏观数据或政策变化

**市场背景**
- 隔夜期货/盘前走势
- 行业ETF表现
- 相关大宗商品或汇率变动
- 今日关键经济数据发布

### 第二步：晨会纪要格式

保持简洁——晨会纪要应在2分钟内可读完：

---

**[日期] 晨会纪要 — [分析师姓名]**
**[覆盖行业]**

**首要推荐：[标题——基金经理最需要知道的一件事]**
- 2-3句话说明关键动态及其重要性
- 股票影响：目标价、评级维持/变化

**隔夜/盘前动态**
- [公司A]：财报/新闻一行摘要 + 我们的观点
- [公司B]：一行摘要 + 我们的观点
- [行业/宏观]：相关的行业整体动态

**今日关键事件**
- [时间]：[公司] 财报电话会议
- [时间]：经济数据发布（预期 vs 我们的观点）
- [时间]：会议或投资者日

**交易创意**（如有）
- [做多/做空] [公司]：1-2句话论点 + 催化剂
- 风险：什么会使这个判断错误

---

### 第三步：财报快速反应

如果覆盖公司发布了财报，提供快速反应：

| 指标 | 一致预期 | 实际值 | 超出/不及 |
|------|---------|--------|----------|
| 收入 | | | |
| EPS | | | |
| [关键指标] | | | |
| 指引 | | | |

**我们的观点**：2-3句话——这对股票是好是坏？是否改变我们的论点？

**行动**：维持/上调/下调评级？调整目标价？

### 第四步：输出

- 用于邮件/Slack分发的Markdown文本
- 如需正式分发则为Word文档
- 最多1页——基金经理和交易员不会读更多

## 重要提示

- 要有观点——只总结新闻而没有看法的晨会纪要毫无用处
- 把最重要的事情放在前面——不要埋没标题
- "没有新闻"也是有效的晨会纪要——说明"隔夜无重大事件，维持持仓"
- 区分可执行事件（财报、并购）和噪音（小幅分析师评级调整、非事件）
- 注明观点时间戳——如果你在早上6点写作，注意盘前可能在开盘前变化
- 如果你判断错了，在第二天的晨会纪要中承认——信誉比每次都正确更重要
