# 股票研究财报更新

创建专业的**财报更新报告**，分析已覆盖公司的季度业绩，遵循机构标准（摩根大通、高盛、摩根士丹利格式）。

**关键特征：**
- **长度**：8-12页
- **字数**：3,000-5,000字
- **表格**：1-3个摘要表（非全面性）
- **图表**：8-12张
- **周转时间**：1-2天（财报发布后24-48小时内）
- **读者**：已熟悉该公司的客户
- **聚焦**：新信息——超预期/不及预期、更新预估、论点影响
- **字体**：全文使用Times New Roman（除非用户另有指定）

## 何时使用

在用户要求以下内容时使用：
- "为[公司] Q3 2024创建财报更新"
- "分析[公司]的季度业绩"
- "[公司]的财报后报告"
- "[公司]的Q1/Q2/Q3/Q4更新"

**不使用的情况：**
- 用户要求"首次覆盖报告" → 使用不同技能
- 用户要求"快讯"或"快速点评" → 不同格式
- 公司尚未被覆盖 → 需先进行首次覆盖

## 关键要求

### 1. 速度与时效性
- 在财报发布后24-48小时内发布
- 仅聚焦新信息
- 不要大量重复公司背景

### 2. 超预期/不及预期分析
- 以公司是否超出或未达预期作为开头
- 量化差异（例如，"收入超出1.2亿元或3%"）
- 解释为什么业绩与预期不同

### 3. 摘要格式
- 表格控制在1-3个（仅摘要，非全面性）
- 不包含完整利润表/现金流量表/资产负债表（仅关键指标）
- 假设读者已看过首次覆盖报告

### 4. 引用与来源标注 ⭐⭐⭐ 必须

**关键要求**：所有数据必须标注具体来源并附可点击超链接。

**每个图表和表格都必须包含带可点击链接的具体引用：**

```
来源：2024年第三季度报告，2024年11月8日发布；公司业绩公告
        [超链接"季度报告"至: https://www.cninfo.com.cn/new/disclosure/detail?...]
        [Hyperlink "earnings release" to: https://investor.company.com/news/q3-2024]
```

**超链接在Word中的呈现方式：**
- 文档名称显示为蓝色、带下划线的可点击链接
- 读者可Ctrl+点击直接打开来源
- 不是纯文本URL——带显示文本的格式化超链接

**必需来源清单：**

每份财报更新必须引用：
- 财报发布（附日期和URL）
- 季度报告（附发布日期和公告查询链接）
- 财报电话会议记录（附日期）
- 投资者演示/补充材料（如有）
- 一致预期来源（Wind / iFinD等，附日期）
- 此前指引（来自上季度材料）

**附可点击超链接的参考文献部分：**

在报告末尾包含"来源"部分：

```
SOURCES & REFERENCES

Earnings Materials (Q3 2024):
• Earnings Release (November 7, 2024)
  [Hyperlink entire line to: https://investor.company.com/news/q3-2024-earnings]

• 季度报告 (2024年11月8日发布)
  [超链接至: https://www.cninfo.com.cn/new/disclosure/detail?...]

• Earnings Call Transcript (November 7, 2024)
  [Hyperlink to: https://seekingalpha.com/article/...]

• Investor Presentation (November 7, 2024)
  [Hyperlink to: https://investor.company.com/presentations/q3-2024.pdf]
```

**验证清单：**
- 每个图表都有附具体文档和日期的来源
- 每个表格都有附文档引用的来源
- 超预期/不及预期分析引用附日期的一致预期来源
- 指引变化引用当前和此前指引来源
- 关键统计数字有脚注
- 来源部分列出所有附URL的材料
- 所有URL均为可点击超链接（非纯文本）
- 所有上市公司公告文件均链接至巨潮资讯网

### 5. 更新预估
- 基于业绩更新前瞻预估
- 清晰展示新旧预估对比
- 解释变化内容和原因

## 高层工作流程

财报更新流程分为5个阶段：

### 阶段一：数据收集（30-60分钟）

**🚨🚨🚨 关键：训练数据已过时 🚨🚨🚨**

**开始前——按顺序完成以下4个步骤：**
1. **检查今天日期**——写下当前日期
2. **搜索最新数据**——使用网络搜索："[公司] latest earnings results"
3. **验证日期**——确认财报发布在最近3个月内
4. **检查记录日期**——验证记录日期与发布日期匹配

**常见错误**：使用训练数据中的过时财报电话会议记录，而非搜索最新数据。

**要求：**
- 搜索最新财报——不要依赖训练数据
- 写下今天日期和找到的发布日期
- 验证发布日期在今天3个月内
- 验证记录日期与发布日期匹配
- 如果日期不匹配或过旧（>3个月），重新搜索

**详见 [references/workflow.md](references/workflow.md)** 了解详细搜索流程和验证步骤。

### 阶段二：分析（2-3小时）
- 每个关键指标的超预期/不及预期分析
- 分部/地域/产品细分
- 利润率和指引分析
- 更新财务模型和预估

**详见 [references/workflow.md](references/workflow.md)** 了解详细分析框架。

### 阶段三：图表生成（1-2小时）
创建8-12张图表，聚焦季度趋势和新变化：
- 季度收入走势
- 季度EPS走势
- 季度利润率趋势
- 按分部/地域的收入
- 关键运营指标
- 超预期/不及预期摘要
- 预估修正
- 估值图表

**详见 [references/workflow.md](references/workflow.md)** 了解图表规格。

### 阶段四：报告创建（2-3小时）
创建8-12页DOCX报告，采用特定结构。

**详见 [reports/report-structure.md](references/report-structure.md)** 了解逐页模板和格式要求。

**高层结构：**
- 第1页：财报摘要，含评级和目标价
- 第2-3页：详细业绩分析
- 第4-5页：关键指标和指引
- 第6-7页：更新投资论点
- 第8-10页：估值和预估
- 第11-12页：附录（可选）

### 阶段五：质量检查与交付（30分钟）
交付前验证内容、格式、准确性和时效性。

**详见 [references/best-practices.md](references/best-practices.md)** 了解质量清单和常见错误。

## 输出规格

**主要交付物**：DOCX报告（8-12页）
**文件名**：`[Company]_Q[Quarter]_[Year]_Earnings_Update.docx`
**示例**：`Nike_Q2_FY24_Earnings_Update.docx`

**内容：**
- 第1页：摘要，含评级、目标价、关键要点
- 第2-3页：详细业绩分析
- 第4-5页：关键指标和指引
- 第6-7页：更新论点评估
- 第8-10页：估值和预估
- 第11-12页：附录（可选）
- 8-12张嵌入图表
- 1-3个摘要表
- 完整来源部分，附可点击超链接

**可选交付物**：XLS模型更新（财报更新为可选）

## 与首次覆盖报告的主要区别

| 方面 | 财报更新 | 首次覆盖报告 |
|------|---------|------------|
| **长度** | 8-12页 | 30-50页 |
| **字数** | 3,000-5,000 | 10,000-15,000 |
| **表格** | 1-3个摘要 | 12-20个全面性 |
| **图表** | 8-12 | 25-35 |
| **周转时间** | 1-2天 | 3-6周 |
| **范围** | 季度业绩 | 完整公司 |
| **聚焦** | 新信息 | 全部 |
| **公司背景** | 简要提及 | 6-10页 |
| **XLS模型** | 可选 | 必须 |
