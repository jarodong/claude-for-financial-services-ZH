**日期**:2026-05-26

**范围**:3 个核心 Agent 系统提示词

---

## 一、本轮工作目标

对 Anthropic 金融套件中已完成初译的 3 个核心 Agent 系统提示词,逐项核对中文译文与英文原版的业务语义一致性,重点检查:

1. 金融行业术语是否对齐国内基金/合规/财务岗实际用语
2. KYC/AML 表述是否符合人民银行、外管监管语境
3. 风险与责任边界表述是否稳妥、不弱化、不夸大
4. 中文全角标点使用是否一致

### 2. 修正成果(共 10 处)

#### valuation-reviewer.md(3 处,高优先级)

| 原译文 | 修正后 | 修正理由 |
|---|---|---|
| 瀑布图(2 处) | **分配瀑布** | "瀑布图"是数据可视化术语;PE/VC 行业 "Waterfall" 指 LP/GP 收益分配机制,国内基金业通用"分配瀑布" |
| 报告标记 | **报告估值** | "marks" 在基金会计语境指"估值",非"标记" |
| 交易时承销 | **交易时的投资评估** | "underwriting" 在 PE 语境是"投资评估/投资决策",非"承销"(承销是债券/IPO 用语) |

#### kyc-screener.md(4 处,中优先级)

| 原译文 | 修正后 | 修正理由 |
|---|---|---|
| 公司 KYC(2 处) | **本机构 KYC** | "公司"在金融机构内部话语中不规范,合规口径用"本机构" |
| 升级(多处) | **上报** | "escalation" 在合规语境是"上报",非"升级" |
| 负面媒体 | **负面舆情** | 国内反洗钱法定术语 |
| 合规签署 | **合规签批** | 国内合规审批口径 |
| PEP | **PEP(政治公众人物)** | 首次出现加注法定中文译名 |
| escalator | **escalator(上报员)** | 加注中文角色名 |

#### gl-reconciler.md(3 处,中优先级)

| 原译文 | 修正后 | 修正理由 |
|---|---|---|
| 系统漂移 | **系统口径差异** | "system drift" 在总账对账语境指"系统口径差异",基金会计岗实操用语 |
| 签署(2 处) | **签批** | 国内财务审批口径 |
| 账簿过账 / 账簿调整 | **入账 / 账务调整** | 国内基金会计/财务共享中心日常用语 |

- **kyc-screener.md 标点不一致**:首次编辑误引入半角 `)` 和 `,`,违反文件全角标点统一规范;立即重读检测到错误后追加修正编辑

## 三、关键判断

### 1. 本轮工作的实质贡献

- **不是文字润色,是业务对齐** — 3 个 Agent 涉及的 PE/VC 估值、KYC/AML 筛查、基金对账三大领域的核心术语全部对齐国内监管和实操口径

## 五、风险与遗留事项

### 主要风险

1. **Skill 层和 subagents 层尚未评审** — Skill 是 Agent 调用的实际工作说明书,subagents 是真正执行写入的叶子节点,这两层的术语和逻辑错误会直接影响 Agent 运行效果,目前完全未核查
2. **partner-built 插件可能存在合规风险** — LSEG 和 S&P Global 涉及境外数据服务条款,中文化时需谨慎处理服务范围与责任表述
3. **术语未沉淀为统一术语表** — 本轮修正的术语对照(如 Waterfall→分配瀑布、escalation→上报)尚未集中归档,后续 7 个 Agent 评审时需要重新人工对照,有不一致风险

### 遗留低优先级事项

- NAV、CCO、IR 等英文缩写首次出现时加中文注释
- "截止日期" → "估值基准日"(基金行业标准表述)
